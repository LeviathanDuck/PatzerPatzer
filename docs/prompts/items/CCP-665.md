# CCP-665 — Global sample size warnings for Prep Report

## Sprint Linkage

- Sprint: SPR-016 (Opponent Research Completion)
- Phase: SPR-016-P2 (Import and Data Integrity)
- Task: SPR-016-T03 (Global sample size warnings)

## Context

The Prep Report displays percentages and statistics across multiple surfaces (win rates,
opening frequencies, deviation rates, etc.). Currently none of these show their sample size,
and small collections produce misleading statistics with no warning. This is a data honesty
pass — every stat must show what it's based on, and unreliable numbers must be visually
distinguished.

## Task

Add sample size indicators to every percentage and statistic in the Prep Report. Implement
a collection-level warning when the total game count is below a configurable threshold.
Visually dim unreliable stats where the denominator is too small for the percentage to be
meaningful.

## Scope

**Files**: `src/openings/view.ts`, `src/openings/analytics.ts` (2 files)

### In `src/openings/analytics.ts`:
- Define a threshold constant for minimum collection size (e.g., `MIN_COLLECTION_SIZE`)
- Define a threshold for minimum stat denominator reliability (e.g., `MIN_RELIABLE_SAMPLE`)
- Add helper functions that analytics consumers can use to check sample reliability
- Ensure every computed statistic carries its sample size (n value) alongside the result

### In `src/openings/view.ts`:
- Render sample size (e.g., "62% (n=13)") next to every percentage in the Prep Report
- When total collection size is below the threshold, show an explicit warning banner
- Apply a visual dim/muted style to stats where the denominator is below the reliability threshold
- Use existing CSS classes or minimal inline styling — do NOT create new CSS files

### Do NOT:
- Change the import pipeline or game parsing
- Modify files outside the two listed files
- Add new analytics computations — only annotate existing ones with sample sizes
