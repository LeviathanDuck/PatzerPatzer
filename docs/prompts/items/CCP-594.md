# Audit: Learn from your mistakes moment selection vs Lichess

## Context

Users have reported that the moments chosen by "Learn from your mistakes" are
sometimes questionable — in particular: a position is flagged as a mistake, but
when the solution is revealed, it shows the same move the user actually played.
This suggests either the candidate selection thresholds, the solution-move
assignment, or the near-best acceptance logic may be off.

## Goal

Produce a written audit comparing Patzer Pro's retrospection implementation
against the Lichess reference, identifying any divergences that could cause
wrong or misleading moments to surface.

## Reference files

**Lichess source** (at `~/Development/lichess-source/lila`):
- `ui/analyse/src/retrospect/retroCtrl.ts` — candidate selection, win/fail logic
- `ui/analyse/src/retrospect/retroView.ts` — feedback rendering

**Patzer Pro source**:
- `src/analyse/retro.ts` — moment candidate extraction, scoring, thresholds
- `src/analyse/retroCtrl.ts` — feedback state machine, win/fail/eval logic
- `src/analyse/retroMoveHandler.ts` — board move interception
- `src/main.ts` — lines around `strictSolutionMove` assignment (search for that term)

## Questions to answer

1. **Solution move assignment** — Patzer sets `strictSolutionMove: node.uci` in
   at least one path in `src/main.ts`. Is `node.uci` the engine's best move or
   the move the user actually played? If it is the user's move, that directly
   explains "solution = move I played."

2. **Candidate scoring** — Does Patzer's win-chance loss calculation in `retro.ts`
   match Lichess's thresholds? What is the minimum WC loss to flag a moment?
   Does Lichess use a different floor?

3. **Near-best acceptance** — `onCeval()` in `retroCtrl.ts` accepts moves above a
   povDiff threshold. What is that threshold, and does it match Lichess's equivalent?

4. **False-positive candidates** — Are there cases where the classification logic
   in `retro.ts` flags a moment where the user's move was actually equal or better?
   (Timing: evaluated at a low depth vs. deeper cached result?)

5. **Depth dependency** — Does Patzer use analysis depth consistently when selecting
   candidates, or could shallow evaluations cause noise?

## Deliverable

A written report covering:
- Each question above: source-confirmed answer, Patzer value, Lichess value, match/diverge.
- A ranked list of divergences by severity (blocks correct behavior / degrades UX / cosmetic).
- Recommended fix prompts for any confirmed bugs (do not implement — log as separate prompts).

## Note

Do not change any code as part of this audit. Research and report only.
