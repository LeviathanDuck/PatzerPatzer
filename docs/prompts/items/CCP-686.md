## Required Review Method

For each child prompt:
1. inspect the prompt body and the work it claims to have completed
2. inspect the actual current code or docs that changed
3. compare the implementation against the prompt contract and current repo workflow
4. run validation only where it materially confirms the child prompt outcome
5. record findings first, then summary
6. close the child prompt with `npm run prompt:review -- ... --reviewed-by Codex --review-method ...`

Manager-specific rules:
- do not mark this manager reviewed while any child above remains unreviewed
- if a child still has live issues, record them honestly and keep the manager blocked until all child reviews are complete
- if a child review requires a follow-up fix prompt, that is allowed, but only after the child review establishes the need
- this manager does not authorize skipping any child review

## Validation

- Confirm every child prompt listed above is no longer in `queued-run` / unreviewed state after the batch review.
- Confirm every child review uses explicit provenance fields:
  - `reviewedBy`
  - `reviewMethod`
- Confirm this manager is reviewed only after all child prompts are reviewed.
- Run `npm run prompts:refresh` after the review batch is closed.

## Lifecycle

Before executing any child prompts, mark this manager as started:
```sh
npm run prompt:start -- CCP-686
```

After all child prompts are reviewed, mark the manager as done:
```sh
npm run prompt:complete -- CCP-686 --checklist "- [ ] All 39 child prompts were reviewed in order|- [ ] Every child review recorded reviewedBy and reviewMethod|- [ ] CCP-686 was reviewed only after all child prompts were reviewed|- [ ] npm run prompts:refresh completed after the batch review"
```

If errors or issues were encountered during execution, use `--errors` instead:

```sh
npm run prompt:complete -- CCP-686 --errors "One or more child reviews could not be completed or the manager could not be honestly closed" --checklist "- [ ] All reachable child prompts were reviewed in order|- [ ] Any blocked child review was recorded honestly|- [ ] CCP-686 remained open or issues-found if child review state was incomplete|- [ ] npm run prompts:refresh completed after the partial batch review"
```
