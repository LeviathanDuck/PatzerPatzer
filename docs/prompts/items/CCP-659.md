# CCP-659 — Fix Opponents Import Cancel and Progress Display

## Context

The Opponents tool shows an import loading overlay with a cancel button when importing a player's
games. Two bugs need to be fixed:

1. **Cancel clears everything** — pressing Cancel currently dismisses the entire overlay AND clears
   all previously imported data. The intended behavior is that Cancel only stops the current import
   in progress; any games already imported should remain.

2. **Missing month in progress display** — the import progress counter currently shows only a game
   number (e.g. "12125 games"). It should also show the month being processed, e.g.
   "12125 games — March 2024", updating as each month's batch is processed.

## Task

Fix both bugs in the opponents import UI. Do not change any other behavior.

### Bug 1: Cancel behavior

- Cancel should stop the ongoing import fetch/processing loop
- Any games already imported before Cancel was pressed must be preserved
- The UI should return to a usable state showing whatever was already loaded
- Do not reset or clear opponent data on cancel

### Bug 2: Month progress label

- While importing, show the month currently being processed alongside the running game count
- Format: `{count} games — {Month} {Year}` (e.g. "12125 games — March 2024")
- Update as each month's batch of games is fetched/processed
- If month info is not available for a given step, fall back to just the game count

## Scope

- Maximum 3 files
- Do not refactor unrelated import logic
- Do not change how games are stored or deduplicated

## Validation

- [ ] Import a player — cancel mid-import — previously loaded games still appear in the tree
- [ ] Import a player — progress label shows "N games — Month YYYY" updating per batch
- [ ] Cancel does not show an error state, just stops cleanly
- [ ] Full import (no cancel) still works correctly end-to-end
