# CCP-647 — Improve sprint dashboard responsiveness and accessibility

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The sprint tab needs design polish, but it also needs a cleaner small-screen and interaction
story. The current card grid, toolbar, pills, and detail content can easily become cramped or
hard to scan on narrower viewports, and the visual polish pass should leave this surface more
robust rather than only nicer on desktop.

## Relevant files

- `scripts/generate-prompt-dashboard.mjs`
- generated output to inspect after changes:
  - `docs/prompts/dashboard.html`

## Goal

Improve the sprint-tab experience for narrower widths and make obvious accessibility and
interaction polish improvements where they are cheap and safe.

Focus on:
- sprint toolbar wrapping and readability
- sprint card behavior on narrower widths
- pill wrapping and density
- detail overlay readability on smaller screens
- obvious focus/interaction polish for sprint-tab controls if needed

## Constraints

- Keep this scoped to responsiveness and accessibility polish for sprint-tab surfaces
- Do not rewrite the sprint data model
- Do not change prompt-tab interaction unless a shared safe improvement is required
- Do not edit generated HTML directly

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html` and verify:
- the `Sprints` tab remains usable at narrower window widths
- toolbar controls wrap cleanly
- pills do not collapse into unreadable clutter
- sprint detail content remains readable on smaller widths
- any added focus or interaction states feel consistent with the rest of the dashboard
