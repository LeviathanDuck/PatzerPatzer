# CCP-619-F1 — Fix: Refresh button hidden at narrow window widths

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

CCP-619 was diagnosed incorrectly. The Refresh Data button disappears at narrow window
widths — this is a responsive layout problem, not a scroll problem.

At ≤700px the header uses `flex-wrap: wrap`. The HTML order is:
  brand → title → stats → generated → refresh

When items wrap, the stats block expands to fill the row and the refresh button gets pushed
to a lower wrapped row where it may be partially or fully hidden.

The fix is to use CSS `order` to keep the refresh button on the first row at narrow widths,
alongside the brand and title, pushed to the right with `margin-left: auto`.

## Target file

`scripts/generate-prompt-dashboard.mjs`

## Changes required

In the existing `@media (max-width: 700px)` block, add three rules:

```css
@media (max-width: 700px) {
  .header { flex-wrap: wrap; height: auto; padding: 8px 12px; gap: 6px; }
  .header__stats { margin-left: 0; flex-wrap: wrap; order: 1; width: 100%; }
  .header__generated { order: 2; }
  .header__refresh { order: 0; margin-left: auto; }
  .container { padding: 10px; }
  .detail-meta { grid-template-columns: 1fr; }
  .detail-meta dt { margin-top: 6px; }
  td.title { max-width: none; }
}
```

The three new lines are:
- `.header__stats` gets `order: 1; width: 100%` — wraps to its own full-width row below the first row
- `.header__generated` gets `order: 2` — sits below stats
- `.header__refresh` gets `order: 0; margin-left: auto` — stays on the first row, pushed to the right after title

Result at narrow widths:
- Row 1: brand | title | [spacer] Refresh Data button
- Row 2: stats (full width)
- Row 3: generated timestamp

### What NOT to change

- Do not change the desktop header layout
- Do not change the header HTML
- Do not touch any other styles outside the existing `@media (max-width: 700px)` block

## Validation

- Narrow the browser to ~400px: Refresh Data button remains visible on the first row, right-aligned
