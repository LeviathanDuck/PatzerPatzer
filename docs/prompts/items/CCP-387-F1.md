## Context

CCP-387 was linked to the opening performance table task but audit evidence shows no opening-specific stats view exists in `src/stats/view.ts`. This follow-up covers Task 9a only — the surface-level opening win rate table built entirely from raw imported game data, with no dependency on analysis (no `GameSummary` required).

Every imported game already carries `result`, `white`, `black`, `eco`, and `opening` fields. No Stockfish review is needed for this table.

## What This Prompt Must Build

In `src/stats/view.ts`, add an opening win rate table section:

**Columns**: Opening name (ECO code as fallback if opening name missing), color played (White / Black), games, W / D / L counts, win rate %

**Rules (from sprint Task 9a)**:
- Aggregate by `opening` field; fall back to `eco` if opening name is missing
- Filter: only show openings with 5+ games played as that color — openings below threshold are hidden
- Separate rows for White and Black (same opening played as both colors = two rows)
- Default sort: worst win rate first
- Highlight rows where win rate is more than 15 percentage points below the user's overall win rate for that color
- Hide the entire table when fewer than 3 qualifying openings (5+ games each)
- Show a subtle "Analyzed data available" indicator on rows that also have `GameSummary` records — this is a future hook for CCP-387-F2 to upgrade those rows with accuracy/blunder columns

**Data source**: the imported games array already in memory — same source the Opening Tree uses. No dependency on `GameSummary` or the analysis pipeline. Table must render immediately after import, before any games are reviewed.

**Rules**:
- Snabbdom `h()` only
- Touch only `src/stats/view.ts`
- No new design system

## Validation

- Opening win rate table renders after game import, before any games are reviewed
- Rows with < 5 games for that opening+color are excluded
- Table hidden when fewer than 3 qualifying opening+color rows
- Rows sorted worst win rate first
- Rows more than 15 points below color average are visually highlighted
- "Analyzed data available" indicator present on rows with matching `GameSummary` records
