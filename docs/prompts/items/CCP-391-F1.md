# CCP-391-F1 — Add Training Recommendation UI to Weakness Cards

## Sprint Linkage

- Sprint: SPR-004 — Mini Sprint — Improvement Intelligence Platform
- Phase: SPR-004-P1 — Tasks
- Task: SPR-004-T14 — Training recommendations
- Source doc: `docs/mini-sprints/IMPROVEMENT_INTELLIGENCE_PLATFORM_SPRINT_2026-03-29.md`
- Fixes: CCP-391 (Add Weakness Training Actions)

## Context

CCP-391 was linked to the training recommendations task. Audit evidence shows that `src/stats/weakness.ts` produces `DiagnosedWeakness` records with a `trainingAction` field, but no UI surfaces these actions — the weakness cards (being completed by CCP-385-F1) render the card structure but the "Train this" action buttons are not wired to navigate anywhere useful.

**Dependency**: CCP-385-F1 (weakness cards) should be complete before this prompt runs, as this prompt wires the action buttons that CCP-385-F1 renders.

## What This Prompt Must Build

In `src/stats/view.ts`, wire each weakness card's training action button to navigate to the correct Patzer tool with appropriate context, per sprint Task 13:

**Action mapping**:

| `trainingAction.type` | `trainingAction.target` | Navigation |
|---|---|---|
| `'puzzles'` | puzzle theme string (optional) | Navigate to `/puzzles` — pass `target` as a theme filter if present |
| `'retro'` | game ID (optional) | Navigate to `/analysis` with the specified game loaded, open retro mode |
| `'openings'` | opening name (optional) | Navigate to `/openings` — highlight the specified opening if possible |
| `'review'` | game ID (optional) | Navigate to `/analysis` with the specified game loaded |

**Rules**:
- Use the app's existing navigation mechanism (match the pattern used in other `view.ts` files for inter-tool navigation)
- If no `trainingAction` is set on a weakness, the card shows no action button (this is the existing CCP-385-F1 behavior; do not change it)
- Each action button label comes from `trainingAction.label` — use it verbatim
- Snabbdom `h()` only
- Touch only `src/stats/view.ts`

## Validation

- "Train this" button on a tactical blindness weakness navigates to `/puzzles`
- "Train this" button on a blunder rate weakness navigates to `/analysis` with retro context
- "Train this" button on an opening weakness navigates to `/openings`
- "Train this" button on a conversion failure weakness navigates to `/analysis` with the game loaded
- Button label matches `trainingAction.label` from the weakness record
- Cards without `trainingAction` show no button
