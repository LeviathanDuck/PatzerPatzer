# CCP-667-F4 — Fix Stylized Editor So It Is Actually Editable

## Summary

Follow up on `CCP-667-F1`.

After entering the stylized editor on prompt detail and sprint detail panels, the resulting
surface is still not behaving like a real editable text editor.

This is distinct from the transition/flicker bug:
- the user may click `Edit`
- the UI may enter what looks like the editor
- but the user still cannot actually edit the text as expected

This prompt is specifically about making the stylized editor truly editable.

## Scope

Fix the actual text-editability of the stylized line-based editor for both:
- prompt detail editing in the `Prompts` tab
- sprint panel editing in sprint detail view

The goal is not to redesign the editor again.

The goal is:
- when the user enters edit mode, the editor behaves like a real editable text surface
- text input, cursor placement, line activation, and text replacement work as intended

## Required Behavior

- prompt detail edit mode must allow real text editing immediately
- sprint panel edit mode must allow real text editing immediately
- activated editable lines must accept typing naturally
- line insertion / replacement behavior must still follow the intended line-based editor model
- locked metadata/header lines must remain non-editable

## Constraints

- keep this scoped to actual text editability of the current stylized editor
- do not redesign the prompt/sprint workflow
- do not remove locked metadata/header behavior
- do not regress copy/save/cancel/reset
- do not regress saved diff/history behavior

## Files Likely Relevant

- `/Users/leftcoast/Development/PatzerPatzer/scripts/generate-prompt-dashboard.mjs`
- `/Users/leftcoast/Development/PatzerPatzer/docs/prompts/dashboard.html`

## Validation

- open an editable prompt and confirm typed text actually changes the active editable line
- open an editable sprint panel and confirm typed text actually changes the active editable line
- confirm locked lines remain non-editable
- confirm save/cancel/reset still work
- confirm prompt detail and sprint detail still render normally outside edit mode
