## What CCP-385 Left Incomplete

`src/stats/view.ts` has section stubs for the weakness panel but does not render:
- Individual weakness cards with severity indicators
- Description and sample-size text per weakness
- Training action buttons/links

## What This Prompt Must Build

In `src/stats/view.ts`, replace the weakness panel placeholder with a fully rendered panel:

**Layout per the sprint spec (Task 7)**:
- Header: "Your Weaknesses" (or "Areas for Improvement")
- If fewer than 10 analyzed games: render "Analyze more games to see your weakness profile" with current game count
- Otherwise: for each `DiagnosedWeakness` (up to 5):
  - Severity indicator — apply a CSS class or inline marker reflecting `severity: 'critical' | 'significant' | 'moderate'`
  - Category label
  - Description text (`weakness.description`)
  - Sample-size context ("Based on N analyzed games")
  - Training action link/button if `weakness.trainingAction` is present ("Train this →")
- Maximum 5 cards; do not render more

**Training action navigation**:
- `type: 'puzzles'` → navigate to `/puzzles` (with `target` as filter if applicable)
- `type: 'retro'` → navigate to the relevant game in `/analysis`
- `type: 'openings'` → navigate to `/openings`
- `type: 'review'` → navigate to `/analysis` with relevant game loaded

**Rules**:
- Snabbdom `h()` only — no direct DOM manipulation
- Follow existing `stats/view.ts` rendering patterns
- No new design system, no chart library
- Touch only `src/stats/view.ts`

## Validation

- Stats page shows real weakness cards (not placeholder text) when 10+ games are analyzed
- Severity class applied to each card (verify via DOM inspection or CSS class presence)
- Insufficient-data state renders the "analyze more games" message with count
