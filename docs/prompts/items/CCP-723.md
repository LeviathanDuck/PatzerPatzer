# CCP-723 — Add Per-Puzzle Engine Output Suppression Reset in LFYM

## Summary

When the user navigates to the next LFYM puzzle, engine output should reset to the
suppressed/hidden state regardless of whether the user had turned it on for the previous puzzle.
Currently `retroVisibleEngineEnabled` persists across puzzle transitions because `jumpToNext()`
only resets `_guidanceRevealed`, not the ceval view flag.

Depends on CCP-722 (unified toggle).

## Task

Reset engine output visibility to suppressed when transitioning between LFYM puzzles.

### Required files to inspect first

- `src/analyse/retroCtrl.ts` — `jumpToNext()` (line 356-370): resets puzzle state
- `src/ceval/view.ts` — `resetRetroVisibleEngineUi()` (line 75-76): resets the flag
- `src/main.ts` — where `jumpToNext()` is called and where `syncArrow()` / `redraw()` follow

### Implementation

1. Ensure that whenever `jumpToNext()` is called (puzzle transition), the engine output
   visibility flag is reset:
   - Call `resetRetroVisibleEngineUi()` from `src/ceval/view.ts` during puzzle transition
   - This should happen in the same place that calls `jumpToNext()` — likely in `src/main.ts`
     or by having `jumpToNext()` accept a callback

2. After resetting, call `syncArrow()` so the arrow shapes immediately reflect the suppressed
   state (no engine arrows on the new puzzle position).

3. Verify that the main Stockfish toggle button visual state also reflects the reset — the
   button should appear as "off" (engine output suppressed) at the start of each new puzzle,
   even though the engine continues running in the background.

### Constraints

- Touch only `src/main.ts` and/or `src/analyse/retroCtrl.ts`
- Do NOT stop or restart the engine — it must continue running in the background
- The reset is rendering-only: the engine keeps analyzing, but output is suppressed
- If the user manually turns on engine output for a puzzle, it stays on until the next
  puzzle transition
- The eval bar should show the "off" state (empty bar) after reset

## Validation

- Start LFYM, turn on engine output for puzzle 1
- Navigate to puzzle 2 — engine output is suppressed (no arrows, PV hidden, eval bar empty)
- Main toggle button appears as "off" for the new puzzle
- User can turn on engine output again for puzzle 2 independently
- Engine continues evaluating in the background during all of this
- npm run build passes
- npx tsc --noEmit passes
