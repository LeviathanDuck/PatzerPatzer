# CCP-642 — Sprint dashboard card layout polish

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The `Sprints` tab in the tracking dashboard is functional, but the card layout currently feels
dense and visually underdesigned. The sprint cards are carrying useful information, but the
current hierarchy makes the content feel closer to a debug panel than a polished dashboard
surface.

The card layout currently combines:
- title and source doc
- status badge
- a three-label progress stack
- summary text
- pills for tasks, prompts, audits, next steps, and dependencies

This needs a stronger visual rhythm, spacing system, and information hierarchy without changing
the underlying sprint semantics.

## Relevant files

- `scripts/generate-prompt-dashboard.mjs`
- generated output to inspect after changes:
  - `docs/prompts/dashboard.html`

## Goal

Improve the sprint card presentation so the `Sprints` tab feels intentional and easier to scan.

Focus on:
- better spacing and grouping within each card
- clearer separation between title/meta, progress, summary, and pills
- more balanced visual treatment for dependency pills and metadata pills
- improved contrast and polish on dark surfaces

## Constraints

- Keep this scoped to sprint card layout and card styling only
- Do not redesign prompt-tab UI
- Do not change sprint filtering, sprint data semantics, or detail-panel structure
- Do not edit generated `docs/prompts/dashboard.html` directly
- Keep the current dark theme direction, but make it feel more designed and less utilitarian

## Design direction

Use design judgment, not just spacing tweaks. The sprint cards should feel more like a purposeful
dashboard component and less like a plain record dump.

Good directions include:
- stronger card section rhythm
- more deliberate use of muted typography tiers
- cleaner grouping of pills into readable clusters
- a more polished top row so the status badge feels integrated instead of floating

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html` and verify:
- sprint cards are easier to scan at a glance
- title, source document, status, progress, summary, and pills feel visually grouped
- dependency pills no longer visually disappear into the rest of the metadata
- no regressions appear in the `Prompts` tab
