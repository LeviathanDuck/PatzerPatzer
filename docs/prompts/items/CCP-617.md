## Context

CCP-616 adds AI Guided Questions gating to prompt creation. This prompt does the same
for sprint creation — but sprints are larger in scope and require a different, expanded
question set covering phases, sequencing, dependencies, and milestone validation.

Sprint creation currently has no structured elicitation step. A sprint is created from
a high-level idea and Claude infers the phases and tasks. This leads to sprints that
miss ordering constraints, skip validation gates between phases, and don't reflect the
user's actual priorities.

This prompt must run AFTER CCP-616 is complete and merged, since it builds on the
CLAUDE.md gate rule already established there.

### CLAUDE.md changes

In the existing **`## AI Guided Questions Mode`** section added by CCP-616, add a
subsection for sprint creation:

**The gate rule for sprints:**
> Before creating any sprint — whether triggered by user request or agent workflow —
> Claude must ask:
>
> *"AI Guided Questions or Quick Create?"*
>
> - **Quick Create**: infer phases and tasks from the request description
> - **AI Guided Questions**: enter sprint guided mode (see below)

**Guided mode behavior for sprints** (expanded vs prompt mode):
- Ask questions one at a time — do not batch
- Sprint-specific questions must cover:
  - **Goal**: What is the single measurable outcome of this sprint?
  - **Scope boundary**: What is explicitly out of scope?
  - **Phases**: Should this be broken into phases? If so, what are they?
  - **Sequencing**: Does any phase depend on another being validated first?
  - **Validation gates**: How do we know each phase is done before moving on?
  - **Prompt breakdown**: Roughly how many prompts will this sprint generate?
  - **Lichess reference**: Is there a Lichess feature this mirrors? Which source files?
  - **Known deviations**: Any intentional departures from Lichess behavior?
  - **Risk**: What is the most likely thing to go wrong in this sprint?
- If an answer is too vague to sequence work from, ask again before continuing
- When all decisions are captured, summarize the proposed sprint structure back:
  *"Here's the sprint structure I'll create — phases, ordering, and key decisions.
  Does this match your intent?"*
- Only after confirmation: proceed to create the sprint document

### SPRINT_CREATION_PROCESS.md

Create or update `docs/mini-sprints/SPRINT_CREATION_PROCESS.md` to document:

```markdown
# Sprint Creation Process

## Creation Modes

Before creating any sprint, Claude must ask:

> "AI Guided Questions or Quick Create?"

**Quick Create** — Claude infers phases and tasks from the request description.
Use only for very small, well-understood sprints.

**AI Guided Questions** — Recommended default for all sprints. Claude asks one
question at a time to establish goal, scope, phases, sequencing, validation gates,
and Lichess reference before writing the sprint document.

## AI Guided Questions — Sprint Question Set

1. What is the single measurable outcome of this sprint?
2. What is explicitly out of scope?
3. Should this be broken into phases? If so, what are they?
4. Does any phase depend on another being validated first?
5. How do we know each phase is done before moving on?
6. Roughly how many prompts will this sprint generate?
7. Is there a Lichess feature this mirrors? Which source files?
8. Any intentional departures from Lichess behavior?
9. What is the most likely thing to go wrong?

Additional questions are asked as needed based on answers.

## Standard Creation Flow

1. Run AI Guided Questions or Quick Create gate
2. Write sprint document in `docs/mini-sprints/`
3. Run `npm run prompts:refresh` to regenerate tracking docs
```
