# Fix retro move feedback lost after offTrack state

## Problem

When the user makes a move during "Learn from your mistakes", instead of receiving
feedback (good move / not best / etc.) the UI immediately shows:

> "You browsed away — Resume learning"

## Root cause

`src/analyse/retroMoveHandler.ts` — the after-move hook gates entry on
`ctrl.retro?.isSolving()`. The `isSolving()` method returns true only when
`_feedback === 'find' || _feedback === 'fail'`.

The timing sequence when a user plays a move from the exercise position:

1. **Before-hook fires** — `feedback = 'find'`, `isSolving()` = true.
   If the move is correct (`uci === bestMove`): `onWin()` sets `feedback = 'win'`.
   If wrong: nothing happens here.
2. **navigate() runs** — path moves from `parentPath` to `parentPath + childId`.
3. **`onJump()` fires inside navigate** — sees `feedback === 'find'` and
   `path !== parentPath` → sets `feedback = 'offTrack'`.
4. **After-hook fires** — `isSolving()` is now **false** (feedback = 'offTrack') →
   hook returns early → no eval state is entered → UI shows offTrack banner.

## Fix

In the after-move hook in `src/analyse/retroMoveHandler.ts`, do not gate on
`isSolving()`. Instead:

- Get `ctrl.retro` directly (not gated on isSolving).
- Still guard: retro must exist, current candidate must exist.
- Skip if `feedback() === 'win'` (correct move already handled by before-hook).
- Perform the `atRetroExercise` path check as before.
- If `atRetroExercise` and `feedback()` is `'offTrack'` (or `'find'`/`'fail'`):
  - If move is wrong: call `retro.setFeedback('eval')` then `retro.onCeval()`.

The before-hook guard on `isSolving()` can remain — it only needs to run during
`'find'` state to pre-set `'win'` before navigate.

## Files to touch (1 file)

- `src/analyse/retroMoveHandler.ts`

## Verification steps

1. Open the analysis board, start Learn from your mistakes.
2. At an exercise position, play a wrong move.
3. Confirm feedback renders (e.g. near-best acceptance or fail state) — NOT "You browsed away".
4. At an exercise position, play the correct move.
5. Confirm win feedback renders as before.
6. Navigate away manually (click a different move in the move list).
7. Confirm "You browsed away / Resume learning" still shows correctly for genuine navigation.
8. No console errors.
