# Fix dead recency toggle in Prep Report

## Problem

`recencyMode()` state exists in `src/openings/ctrl.ts` and is rendered as a toggle in the
Prep Report UI, but toggling "Recent" / "All time" has no visible effect. Two root causes:

1. `setRecencyMode()` does not invalidate `_prepReportCache`, so the view always returns
   the previously cached result.
2. `computeLikelyLineModule()` in `src/openings/analytics.ts` ignores recency mode entirely —
   it always applies recency boosts regardless of which mode is active.

## Acceptance criteria

- Toggling the recency control in Prep Report causes the lines to visibly reorder / change
  frequency display.
- "All time" mode: lines sorted by raw cumulative frequency; no recency boost applied.
- "Recent" mode: lines sorted by recency-boosted score (existing behavior).
- Switching modes re-renders immediately without a page reload.

## Files to touch (max 2)

- `src/openings/ctrl.ts` — invalidate `_prepReportCache` in `setRecencyMode()`
- `src/openings/analytics.ts` — add `recencyMode` parameter to `computeLikelyLineModule()`
  and `computePrepReportLines()`; when mode is `'all-time'` return 1.0 from `recencyBoostFor`
  and sort by raw frequency

## Implementation notes

### ctrl.ts

In `setRecencyMode(mode)`, add `_prepReportCache = null` before or after the existing
state assignment so the next call to `getPrepReport()` recomputes.

Also update `getPrepReport()` (or wherever `computePrepReportLines` is called) to pass
`recencyMode()` as an argument.

### analytics.ts

Change the signature of `computeLikelyLineModule` to accept a `recencyMode: 'recent' | 'all-time'`
parameter (default `'recent'` to preserve existing behavior).

Inside `computeLikelyLineModule`, when `recencyMode === 'all-time'`:
- Replace all `recencyBoostFor(...)` calls (or the inner scorer) with a constant `1.0`
- Sort the final candidates by raw `frequency` (or `cumulativeFreq`) descending instead
  of by the boosted score

Propagate the parameter up through `computePrepReportLines` → caller in `ctrl.ts`.

## Verification steps

1. Navigate to `/openings`, load a collection with enough games.
2. Open the Prep Report tab.
3. With "Recent" selected, note the top lines and their order.
4. Click "All time" — lines should reorder, recency-specific boosts should disappear.
5. Click "Recent" again — order should return to original.
6. Confirm no console errors.
