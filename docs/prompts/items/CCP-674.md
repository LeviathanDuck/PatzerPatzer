# CCP-674 — Update Server Endpoints For Textarea Editor And Unified Archival

Prompt ID: CCP-674
Task ID: CCP-674
Source Document: (audit conversation, 2026-04-02)

## Context

The server endpoints `/api/prompt-edit` and `/api/sprint-panel-note` currently expect payloads
shaped by the old line editor. After CCP-672 (textarea editor) and CCP-673 (unified archival),
the server handlers need to be updated to match the simplified client and the new sprint archival model.

CCP-672 and CCP-673 must be completed first.

## Task

Update the server-side handlers in `server.mjs` to work with the new textarea-based editor
and the unified RETIRED-V# archival for sprint panels.

### `/api/prompt-edit` (handlePromptEditSave)

This endpoint is likely already compatible since it receives `{ promptId, body }` — the textarea
just sends the same payload. However:
1. Verify it still works end-to-end with the new textarea client
2. Confirm the superseded archival still creates RETIRED-PROMPT-DONOTRUN-REFERENCE-ONLY entries
3. Remove any code that referenced line editor state if present
4. Ensure `lastEditedAt` is still set on the active prompt

### `/api/sprint-panel-note` (handleSprintPanelNoteWrite)

This endpoint needs updating to use the new archival pattern from CCP-673:
1. On save, call the updated `sprint-registry-lib` archival function that creates RETIRED-V# entries
2. Stop writing to `panelNotes.history` array (the lib change from CCP-673 handles this)
3. Ensure the response returns the retired entry ID so the client can confirm archival
4. Ensure `lastEditedAt` is still set

### `/api/sprint-panel-note/clear`

Verify that clearing a saved panel note still works correctly after the archival change.
Clearing should not create a retired entry — it just removes the saved override.

### Files to touch

1. `server.mjs` — update handlers

## Validation

1. Start `node server.mjs`
2. Edit and save a prompt from the dashboard — verify server returns `{ok: true, prompt, archivedId}`
3. Edit and save a sprint panel — verify server returns the retired entry ID
4. Clear a sprint panel note — verify it falls back to the generated default
5. Check `prompt-registry.json` for correct RETIRED prompt entry after prompt save
6. Check `sprint-registry.json` for correct RETIRED sprint panel entry after sprint save
7. Verify no references to line editor state remain in server.mjs
