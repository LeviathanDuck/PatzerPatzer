# CCP-510 — Rename "Repertoire" tool to "Opening Tree"; move prep-zone to Repertoire tool

## Context

The left tool rail on the opponent research session page has four tools:
`repertoire`, `prep-report`, `style`, `practice`.

The `repertoire` tool currently renders the board, opening tree move list, and a
`.openings__prep-zone` analytics block (overview, summary modules, line insight
cards). The prep-zone content belongs on a dedicated Repertoire dashboard, not
embedded in the tree navigation view.

The goal of this prompt is to:
1. Rename the top tool from "Repertoire" to "Opening Tree" with a relevant icon
2. Strip the prep-zone block out of the Opening Tree panel (board + move list only)
3. Add "Repertoire" as the second tool in the rail — rendering the prep-zone content
   that was removed from Opening Tree

## Changes

### `src/openings/types.ts`

Change the `OpeningsTool` type:

```typescript
// Before:
export type OpeningsTool = 'repertoire' | 'prep-report' | 'style' | 'practice';

// After:
export type OpeningsTool = 'opening-tree' | 'repertoire' | 'prep-report' | 'style' | 'practice';
```

### `src/openings/ctrl.ts`

- Replace all `'repertoire'` default/reset values with `'opening-tree'` in:
  `openCollection()`, `closeSession()`, and any `initOpeningsPage` call.
- In the session restore block, add `'opening-tree'` to `validTools` and keep
  `'repertoire'` for backwards compatibility with saved sessions that stored
  `'repertoire'` as the active tool (restore those as `'opening-tree'`).

### `src/openings/view.ts`

**`TOOL_DEFS` array** — update to:
```typescript
const TOOL_DEFS: ToolDef[] = [
  { id: 'opening-tree', label: 'Opening Tree', icon: ICON_BRANCH }, // '\ue003' licon.Branch
  { id: 'repertoire',   label: 'Repertoire',   icon: ICON_BOOK },
  { id: 'prep-report',  label: 'Prep Report',  icon: ICON_BAR_GRAPH },
  { id: 'style',        label: 'Style',        icon: ICON_EYE },
  { id: 'practice',     label: 'Practice',     icon: ICON_SWORDS },
];
```

For the Opening Tree icon, use `'\ue003'` (licon.Branch). If that doesn't render,
fall back to `'\ue004'` (licon.AnalBoard).

**`TOOL_NAMES` map** — add `'opening-tree': 'Opening Tree'` entry.

**`renderSessionPage` dispatch block** — update:
```typescript
...(activeTool() === 'opening-tree'
  ? renderOpeningTreeTool(collection, node, path, redraw)
  : activeTool() === 'repertoire'
    ? [renderRepertoireDashboard(collection, redraw)]
    : activeTool() === 'prep-report'
      ? [renderPrepReportTool(redraw)]
      : [renderToolPlaceholder(activeTool())]),
```

**Rename `renderRepertoireTool` → `renderOpeningTreeTool`** and strip the
following from the panel (move them, not delete them):

Remove from `renderOpeningTreeTool`:
- The `openings__repertoire-header` block (label + game count)
- The `openings__prep-zone` block (overview + summary modules + line insight cards)
- The `openings__zone-divider` that follows the prep-zone

The Opening Tree panel should then be: action menu → color toggle → tree build bar
→ played lines → deviation panel → sample games → move list → explorer → engine.

**Add `renderRepertoireDashboard(collection, redraw)`** — a new function that
renders the prep-zone content as a standalone full-width tool view:

```typescript
function renderRepertoireDashboard(
  collection: ResearchCollection | null,
  redraw: () => void,
): VNode {
  return h('div.openings__tool-content', [
    h('div.openings__prep-zone', treeBuilding()
      ? [h('div.openings__prep-zone-loading', 'Building tree\u2026')]
      : [
          collection ? renderRepertoireOverview(collection) : null,
          renderRepertoireSummaryModules(redraw),
          renderLineInsightCards(redraw),
        ]
    ),
  ]);
}
```

Note: `renderRepertoireDashboard` returns a single `VNode` (not `VNode[]`) because
it spans the full content area like `renderPrepReportTool`. The CSS class
`openings__tool-content` already handles this layout.

**Update `setActiveTool` calls** that previously sent the user back to repertoire:
- In `navToLine` inside `renderPrepReportTool`: `setActiveTool('repertoire')` →
  `setActiveTool('opening-tree')`
- In the trap card click handler: same swap
