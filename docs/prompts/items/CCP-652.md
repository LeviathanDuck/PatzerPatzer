# CCP-652 — Design sprint detail action panel shell

Read and follow:

## Context

Codex has implemented the backend contract for sprint-detail action surfaces and added basic
functional rendering to the shared dashboard:
- always-visible sprint audit copy panel
- conditional next-phase prompt creation panel
- normalization warning state
- next available phase derivation
- task-state and sprint-state rollups

The user explicitly wants Claude to own the final UI design decisions for these action surfaces.

## Relevant file

- `scripts/generate-prompt-dashboard.mjs`

Generated output to inspect:
- `docs/prompts/dashboard.html`

## Goal

Design the overall shell and placement for the sprint-detail action-panel area near the top of the
sprint detail view.

This prompt should decide:
- how the action area is framed relative to sprint summary/meta
- how the audit panel and next-prompt panel coexist
- how strong or quiet the action shell should feel

## Hard constraints

- Do not invent new sprint states or new action rules
- Do not change when the audit panel or next-prompt panel appears
- Do not change backend derivation logic
- Keep this prompt scoped to the shared action-panel shell and placement

## Backend contract already implemented

Consume the existing backend fields and visibility rules as-is.

The UI work here should be presentation only.

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html`, open multiple sprint details, and verify:
- the action area feels intentional near the top of sprint detail
- the sprint audit and next-prompt surfaces coexist cleanly
- the shell improves scanability without overwhelming the rest of the detail view
