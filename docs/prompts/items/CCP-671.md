## Context

The dashboard inline editor in `generate-prompt-dashboard.mjs` has a systematic escaping bug.
When the generator writes JS functions into the HTML `<script>` block, it uses `'\\\\n'` (4 backslashes + n)
where it should use `'\\n'` (2 backslashes + n). This means that in the browser, `split('\\n')` tries to
split on literal backslash-n (2 chars) instead of newline (char code 10). The result: the entire prompt
body renders as a single `<input>` element, and locked header lines are concatenated into one row.

The full bug analysis with hex-verified evidence is in `docs/internal/BUG_LINE_EDITOR_NEWLINE_MISMATCH.md`.

## Task

Fix the escaping mismatch in `scripts/generate-prompt-dashboard.mjs` at the specific lines identified
in the bug doc. Then regenerate `docs/prompts/dashboard.html` and verify the editor works.
