# CCP-562: Opening Classification Sprint Manager

## Overview

Manager prompt that executes the full Opening Classification sprint in sequence.

## Child Prompts (execute in order)

1. **CCP-515** — Create ECO classifier module (`src/openings/eco.ts`)
2. **CCP-516** — Wire classification into game import paths (pgn.ts, lichess.ts, chesscom.ts)
3. **CCP-517** — Backfill existing games + wire openings research import
4. **CCP-518** — Add IDB eco index + opening filter in Games tab

## Execution Instructions

Execute each child prompt in strict order. Each depends on the previous.

For each child prompt:
1. Run `npm run prompt:start -- <CCP-ID>`
2. Read the prompt file at `docs/prompts/items/<CCP-ID>.md`
3. Implement the task described in the prompt
4. Verify with `npm run build` and `npx tsc --noEmit`
5. Run `npm run prompt:complete -- <CCP-ID> --checklist "..."` with appropriate verification checklist
6. Report: prompt ID, build result, typecheck result, whether batch continues or stops

If a child prompt fails build or typecheck, fix the issue before moving to the next prompt. If the issue cannot be resolved, stop the batch and report which prompt failed and why.

Do not reorder child prompts. Do not create new prompts during the batch. Do not skip prompts.

## Sprint Context

This sprint adds local opening name/ECO classification for all imported chess games. The data source is the lichess-org/chess-openings TSV database (~3,500 entries). Games are classified by replaying moves with chessops and matching positions against the static ECO database.

## Final Validation

After all 4 child prompts complete, verify end-to-end:
- Import a PGN via paste (no Opening/ECO headers) → opening column shows classified name
- Import games via Lichess/Chess.com username → all games have opening names
- Existing games that previously showed "–" now show classified names after app reload
- Games tab has an opening filter input that filters by name or ECO code
- `npm run build` passes
- `npx tsc --noEmit` passes
