# CCP-680 — Learn From Your Mistakes: Eval Diff Showing Blank

## Summary

In Learn From Your Mistakes mode, the eval diff between the move played in the game and the best move is displaying as blank. Find the source of the missing value and fix it so the eval diff renders correctly.

## Context

The Learn From Your Mistakes surface is expected to show the evaluation difference between:
- the move the user played in the game
- the best move available at that position

This delta is used to communicate how much was lost by playing the inferior move. It is currently rendering as blank/empty instead of showing the centipawn or mate-distance value.

## Tasks

1. Locate the code responsible for computing or rendering the eval diff in the Learn From Your Mistakes view
2. Determine whether the value is:
   - not being computed (missing logic)
   - computed but not passed through to the view
   - passed through but formatted/rendered incorrectly (e.g. undefined, NaN, empty string)
3. Fix the root cause so the eval diff value displays correctly

## Scope

- Touch only the files involved in computing and rendering the eval diff in Learn From Your Mistakes
- Do not refactor the surrounding analysis or engine layers
- Maximum 1–3 files

## Validation

- [ ] Eval diff between move played and best move is visible and non-blank in Learn From Your Mistakes
- [ ] Value is correct (centipawns or mate notation, matching what the engine reported)
- [ ] No regression in surrounding Learn From Your Mistakes feedback (move quality label, solution reveal, etc.)
