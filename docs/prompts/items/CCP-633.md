## Context

Part 2 of 4 in the retro live eval diff feature (manager: CCP-636).
Depends on CCP-632 being complete.

When the user plays a move during a retro puzzle, the engine's current best move and
eval must be captured at that moment and stored alongside the solving move's eval.
Two diffs are then computed (both mover-perspective, in pawn units):

1. Solving move vs engine best (how far off from perfect)
2. Solving move vs original game move (was the attempt at least better than the game?)

This prompt wires the engine feed into `RetroCtrl.onEngineUpdate()` and captures the
solving move eval snapshot at move-play time.

## Target files

- `src/analyse/retroMoveHandler.ts` — capture solving move eval at move time
- `src/main.ts` — feed engine updates to `retro.onEngineUpdate()`

## Sign convention (critical)

All diffs must be **mover-perspective**: positive = better for the player being reviewed.

For a white candidate:
- `moverCp = +cp` (white-perspective cp is already mover-perspective)

For a black candidate:
- `moverCp = -cp` (flip sign so positive = better for black)

The diff between two moves A and B (from mover's perspective):
- `diff = moverCp(A) - moverCp(B)`
- Positive diff = A is better for the mover than B

## Dual eval diff type

Add to `src/analyse/evalDiff.ts` (or a shared types location):

```typescript
export interface DualEvalDiff {
  /** Solving move vs engine best. Negative = worse than best. */
  vsEngineBest: EvalDiff;
  /** Solving move vs original game move. Positive = better than game move. */
  vsGameMove: EvalDiff;
}
```

are processed for the active position), after updating the eval cache, call:

```typescript

  ctrl.retro.onEngineUpdate(bestUci, { cp, mate, depth });
```

Only call this when the update is for the active retro candidate's position

### src/analyse/retroMoveHandler.ts

When the user plays a move during solving (the existing move-capture path), after
determining the feedback state, add:

```typescript
// Snapshot the engine's current best and eval at the moment of the move.
// Use whichever depth is higher: live (via retro.liveBestEval) or batch (cand.bestMove).
const liveBest = retro.liveBestMove() ?? cand.bestMove;
const liveEval = retro.liveBestEval();
const batchDepth = 0; // batch has no stored depth
const useLive = liveEval && liveEval.depth > batchDepth;

// Store the solving move's position eval and the snapshotted best for CCP-634 to render.
retro.setSolvingMoveSnapshot({
  solvingMoveUci: playedUci,
  solvingMoveCp:  /* eval of the position after the solving move, from evalCache */,
  engineBestUci:  liveBest,
  engineBestCp:   useLive ? liveEval.cp : /* batch best eval cp */,
  gameMoveUci:    cand.bestMove,  // original game move (the mistake)
  gameMoveCp:     /* eval of the position after the original game move, from evalCache */,
  playerColor:    cand.playerColor,
});
```

Add `setSolvingMoveSnapshot(snapshot: SolvingMoveSnapshot): void` to `RetroCtrl`
(defined in CCP-632's interface additions).

Add `SolvingMoveSnapshot` type:
```typescript
export interface SolvingMoveSnapshot {
  solvingMoveUci: string;
  solvingMoveCp:  number | undefined;
  solvingMoveMate: number | undefined;
  engineBestUci:  string;
  engineBestCp:   number | undefined;
  engineBestMate: number | undefined;
  gameMoveUci:    string;
  gameMoveCp:     number | undefined;
  gameMoveMate:   number | undefined;
  playerColor:    'white' | 'black';
}
```

Expose `getSolvingMoveSnapshot(): SolvingMoveSnapshot | undefined` on `RetroCtrl`.
Clear snapshot on `jumpToNext()` / `reset()`.
