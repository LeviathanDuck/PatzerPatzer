# CCP-393-F1 — Add Conversion and Resourcefulness Metrics to Stats Page

## Sprint Linkage

- Sprint: SPR-004 — Mini Sprint — Improvement Intelligence Platform
- Phase: SPR-004-P1 — Tasks
- Task: SPR-004-T16 — Conversion and resourcefulness
- Source doc: `docs/mini-sprints/IMPROVEMENT_INTELLIGENCE_PLATFORM_SPRINT_2026-03-29.md`
- Fixes: CCP-393 (Add Conversion And Resourcefulness Metrics)

## Context

CCP-393 was linked to the conversion and resourcefulness task. Audit evidence shows that `GameSummary` records (from `src/stats/types.ts`) include `hadWinningPosition`, `converted`, `hadLosingPosition`, and `survived` boolean fields, and that these are computed during extraction. However no stats view renders these metrics.

## What This Prompt Must Build

In `src/stats/view.ts`, add a conversion and resourcefulness section per sprint Task 15:

**Conversion rate** (shown when at least 5 games had `hadWinningPosition: true`):
- "You reached a winning position in X games and converted Y of them (Z%)"
- `X` = count of games where `hadWinningPosition: true`
- `Y` = count of games where `converted: true`
- `Z` = Y / X as a percentage

**Resourcefulness rate** (shown when at least 5 games had `hadLosingPosition: true`):
- "You were in a losing position in X games and saved Y of them (Z%)"
- `X` = count of games where `hadLosingPosition: true`
- `Y` = count of games where `survived: true`
- `Z` = Y / X as a percentage

**Failed-conversion game list** (stretch goal — implement if straightforward):
- List the specific games where `hadWinningPosition: true` AND `converted: false`
- Each entry is a clickable link that loads the game in `/analysis`
- Limit to the 5 most recent failed conversions
- Label: opponent name + date

**Show/hide rules**:
- Conversion rate hidden when fewer than 5 games had a winning position
- Resourcefulness rate hidden when fewer than 5 games had a losing position
- If both are hidden, the entire section is absent — no placeholder

**Rules**:
- Snabbdom `h()` only
- Touch only `src/stats/view.ts`
- No chart library
- No new design system

## Validation

- Conversion rate sentence renders when 5+ games had a winning position
- Resourcefulness rate sentence renders when 5+ games had a losing position
- Correct counts and percentages computed from `GameSummary` fields
- Section completely absent when neither threshold is met
- Failed-conversion game links navigate to `/analysis` with the correct game loaded (if stretch goal implemented)
