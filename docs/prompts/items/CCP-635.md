### Individual eval box

Each box has a label on the left and a value badge on the right.
Four color tiers: `--best` (green), `--near` (yellow), `--ok` (blue), `--bad` (red).

```scss
.retro-eval-box {
  display: flex;
  align-items: center;
  justify-content: space-between;
  padding: 5px 10px;
  border-radius: 4px;
  border: 1px solid transparent;

  &__label {
    font-size: 0.78rem;
    color: #888;
  }

  &__value {
    font-size: 0.85rem;
    font-weight: 700;
    font-variant-numeric: tabular-nums;
    min-width: 2.5em;
    text-align: right;
  }

  // Green — exact best move
  &--best {
    border-color: #3a6b2a;
    background: rgba(117, 153, 0, 0.12);
    .retro-eval-box__value { color: #759900; }
  }

  // Yellow — slightly worse than best (0 to -0.5 pawns)
  &--near {
    border-color: #7a6000;
    background: rgba(240, 165, 0, 0.1);
    .retro-eval-box__value { color: #f0a500; }
  }

  // Blue — better than game move, more than 0.5 pawns from best
  &--ok {
    border-color: #2a4a7a;
    background: rgba(56, 100, 160, 0.1);
    .retro-eval-box__value { color: #7aaddf; }
  }

  // Red — worse than or equal to original game move
  &--bad {
    border-color: #6b2a2a;
    background: rgba(220, 50, 47, 0.1);
    .retro-eval-box__value { color: #dc322f; }
  }
}
```

### What NOT to change

- Do not change any existing retro rules.

## Validation

Run `npm run build`. Open a reviewed game, click Mistakes, play moves.
- Green box with ✓ when solving move matches engine best.
- Yellow box when within 0.5 pawns of best.
- Blue box when better than game move but clearly not best.
- Red box when worse than or equal to the original game move.
- Vindication notice styled with green left border when it appears.
