# CCP-657 — Curated Master Games Dataset: Fischer and Capablanca

## Context

The app needs a static dataset of ~200 iconic chess games by Bobby Fischer and José Raúl
Capablanca. These games will be embedded in the app as a bundled TypeScript module and used
as showcase/sample games wherever a "demo board" is needed — most immediately as the source
for the import animation (CCP-658), but reusable for any future board showcase context.

All games by both players from this era are public domain in the United States.

## Task

### 1. Create the game data file

Create `src/showcase/masterGames.ts`.

Define a `MasterGame` interface:

```typescript
export interface MasterGame {
  id:       string;          // e.g. 'fischer-001'
  white:    string;          // full name
  black:    string;          // full name
  result:   '1-0' | '0-1' | '1/2-1/2';
  year:     number;
  event:    string;          // tournament or match name
  site:     string;          // city / venue if known, else ''
  eco:      string;          // ECO code if known, else ''
  opening:  string;          // opening name if known, else ''
  label:    string;          // one-line human display: "{White} vs {Black}, {Event} {Year}"
  moves:    string[];        // UCI move array, e.g. ['e2e4', 'e7e5', 'g1f3', ...]
}
```

### 2. Curate the games

Include approximately 200 games split roughly evenly between Fischer and Capablanca.

**Fischer selection (~100 games):**
Include his most famous and instructive games. Prioritize:
- Game of the Century (1956, Fischer vs Donald Byrne)
- Match of the Century games vs Spassky (1972 World Championship)
- Fischer vs Reshevsky matches
- US Championship games
- Palma de Mallorca Interzonal 1970 (20/23 score)
- Candidates matches (Taimanov, Larsen, Petrosian)
- Memorable wins in Sicilian, King's Indian, Nimzo-Indian

**Capablanca selection (~100 games):**
- World Championship 1921 (vs Lasker)
- World Championship 1927 (vs Alekhine)
- New York 1924 tournament
- San Sebastian 1911
- London 1922
- Notable endgame masterpieces (Capablanca was renowned for endgames)
- Famous immortal games and sacrifices

### 3. Move encoding

Store moves as UCI strings (e.g. `'e2e4'`, `'g1f3'`, `'e1g1'` for kingside castle).

Use chessops to verify moves are legal before including them:
- Parse starting FEN (standard start position for all these games)
- Walk through each move list using `Chess`, `parseUci`, `makeMove`
- Skip any game where a move is illegal (log to console during generation)

This verification step ensures no corrupt move sequences enter the dataset.

### 4. Module exports

```typescript
export const MASTER_GAMES: MasterGame[] = [ ... ];

/** Return a random game from the full dataset. */
export function randomMasterGame(): MasterGame {
  return MASTER_GAMES[Math.floor(Math.random() * MASTER_GAMES.length)]!;
}

/** Return a random game by a specific player (matched against white or black field). */
export function randomGameByPlayer(name: string): MasterGame | undefined {
  const matches = MASTER_GAMES.filter(
    g => g.white.toLowerCase().includes(name.toLowerCase()) ||
         g.black.toLowerCase().includes(name.toLowerCase()),
  );
  if (matches.length === 0) return undefined;
  return matches[Math.floor(Math.random() * matches.length)]!;
}
```

### 5. File size budget

The move arrays are compact (each UCI string is 4–5 chars). 200 games × ~40 moves average
≈ 8000 strings ≈ ~60–80 KB unminified, well within the existing 1.4 MB bundle budget.
