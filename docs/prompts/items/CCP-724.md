# CCP-724 — Add Try Another Move Button to LFYM Retro Panel

## Summary

After the user fails a puzzle attempt in LFYM, the only options are Skip, View Solution, or
continue trying from the current board state. There should be a "Try another move" button that
resets to the puzzle start position and re-suppresses engine output, allowing a clean retry.

Depends on CCP-722 (unified toggle) and CCP-723 (suppression reset).

## Task

Add a "Try another move" button to the LFYM retro panel that appears in the `fail` feedback state.

### Required files to inspect first

- `src/analyse/retroView.ts` — `fail` feedback block (lines ~515-540): where the button should go
- `src/analyse/retroCtrl.ts` — puzzle state, `jumpToNext()` pattern for state reset
- `src/ceval/view.ts` — `resetRetroVisibleEngineUi()` for engine suppression reset
- `src/main.ts` — navigation callback (`navigate`) used by existing retro panel buttons

### Implementation

1. In `src/analyse/retroCtrl.ts`:
   - Add a `retryPuzzle()` method that:
     - Resets `_feedback` to `'find'`
     - Resets `_failKind` to `null`
     - Resets `_winKind` to `null`
     - Does NOT call `solveCurrent()` — the puzzle stays unsolved
     - Does NOT advance `currentIdx` — stays on the same puzzle

2. In `src/analyse/retroView.ts`:
   - In the `fail` feedback block, add a "Try another move" button after the existing
     "Try another move for {Color}" instruction text
   - The button click handler should:
     - Call `retro.retryPuzzle()` to reset feedback state
     - Call `resetRetroVisibleEngineUi()` to re-suppress engine output
     - Navigate back to the candidate's `parentPath` (puzzle start position)
     - Call `syncArrow()` and `redraw()`
   - Style the button consistently with existing retro panel buttons (use `retro-choices` pattern)

3. In `src/main.ts`:
   - Wire the `retryPuzzle` callback through to the retro view if it's not already accessible
     through the retro object passed to `renderRetroBox()`

### Constraints

- Touch only `src/analyse/retroCtrl.ts`, `src/analyse/retroView.ts`, and `src/main.ts` if needed
- The button must only appear in the `fail` feedback state
- Do NOT change Skip or View Solution behavior
- The retry must navigate back to the puzzle start position (`cand.parentPath`)
- Engine output must be re-suppressed on retry (same as new puzzle behavior)
- The puzzle must remain unsolved — the user gets another attempt

## Validation

- Start LFYM, make an incorrect move
- "Try another move" button appears in the fail feedback
- Clicking it returns to the puzzle start position
- Engine output is re-suppressed (arrows hidden, eval bar empty)
- The puzzle is not marked as solved — user can try again
- Skip and View Solution still work normally alongside the new button
- npm run build passes
- npx tsc --noEmit passes
