# CCP-631 — Retro: style retro-reason label in learn from mistakes feedback

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The Learn From Your Mistakes feedback box renders several elements that have no CSS rules:
- `.retro-reason` — "Chosen because: Blown win" label + summary sentence
- `.retro-reason__label` — the short label line
- `.retro-reason__summary` — the one-sentence explanation
- `.retro-feedback__eval-diff` — the eval diff badge (e.g. "+1.2", "M3")
- `.retro-save` / `.retro-save__btn` / `.retro-save__done` / `.retro-save__confirm` — save to library button and states

All of these render invisible or completely unstyled. This prompt adds appropriate styles
for all of them inside the existing `.retro-box { }` block in `src/styles/main.scss`.

## Target file

`src/styles/main.scss` only.

## Insertion point

Inside `.retro-box { }`, before the closing `}` at line ~1689 (just after `.retro-engine-reveal`).

## Changes required

### .retro-reason

A small callout block that appears after the eval diff in the win/view feedback states.
Should be visually distinct but quiet — not competing with the main "Good move!" heading.
Use a subtle left border accent to visually group it.

```scss
.retro-reason {
  margin-top: 10px;
  padding: 7px 10px;
  border-left: 3px solid #3a5a8a;
  background: rgba(56, 100, 160, 0.1);
  border-radius: 0 4px 4px 0;
  display: flex;
  flex-direction: column;
  gap: 2px;
}

.retro-reason__label {
  font-size: 0.8rem;
  font-weight: 600;
  color: #7aaddf;
}

.retro-reason__summary {
  font-size: 0.78rem;
  color: #888;
  line-height: 1.4;
}
```

### .retro-feedback__eval-diff

A compact badge shown inline in the instruction column between the heading and the reason note.
Should stand out as a numeric value without dominating.

```scss
.retro-feedback__eval-diff {
  display: inline-block;
  margin-top: 4px;
  font-size: 0.85rem;
  font-weight: 600;
  color: #dc322f;
  font-variant-numeric: tabular-nums;
}
```

### .retro-save family

The save button and its state variants. The button should match the quiet `.fbt` style
(already used elsewhere in the retro box for nav buttons) but with a subtle accent.

```scss
.retro-save {
  margin-top: 10px;
}

.retro-save__btn {
  font-size: 0.8rem;
  padding: 4px 12px;
  border-radius: 4px;
  border: 1px solid #3a5a8a;
  background: transparent;
  color: #7aaddf;
  cursor: pointer;
  font-family: inherit;
  transition: background 0.15s, color 0.15s;

  &:hover {
    background: rgba(56, 100, 160, 0.2);
    color: #a8ccee;
  }
}

.retro-save__done {
  font-size: 0.8rem;
  color: #666;
}

.retro-save__confirm {
  font-size: 0.8rem;
  color: #759900;
  font-weight: 600;
}
```

### What NOT to change

- Do not change any retro layout, feedback states, or existing retro rules
- Do not change `.retro-bulk-save` if it already has styles (check first)
- Do not touch any files other than `src/styles/main.scss`

## Validation

Run:
```
npm run build
```

Open a reviewed game, click Mistakes, complete a candidate. Verify:
- The "Chosen because: ..." block is visible with blue-left-border styling
- The label line is readable (light blue, semi-bold)
- The summary line is visible (muted grey)
- The eval diff badge (e.g. "+1.2") is visible in red above the reason block
- The Save to Library button is visible and styled
- "Saved!" confirmation appears in green when clicked
- No visual regression on other retro feedback states (find, fail, offTrack, eval)
