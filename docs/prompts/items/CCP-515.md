# CCP-515: Create ECO Classifier Module

## Goal

Create `src/openings/eco.ts` — a self-contained module that maps a game's move sequence to its deepest-matching ECO opening using a static database vendored from the lichess-org/chess-openings repository.

## Data Source

Fetch TSV files from https://github.com/lichess-org/chess-openings (a.tsv through e.tsv). Each row has: ECO code, opening name, PGN moves, UCI moves, and EPD (position-only FEN).

## Implementation

1. **Fetch the TSV data** from the lichess-org/chess-openings repo (raw GitHub URLs for a.tsv–e.tsv).

2. **Convert to a TypeScript map** keyed by normalized FEN (first 4 FEN fields — piece placement, active color, castling, en passant — matching the `normalizeFen` pattern in `src/openings/tree.ts:82-87`). The map value is `{ eco: string; name: string }`.

3. **Embed the data as a `const` literal** in `src/openings/eco.ts`. The ~3,500 entries at ~100 bytes each is roughly 350KB source, which compresses well and avoids async loading.

4. **Export `classifyOpening(pgn: string): { eco: string; name: string } | undefined`**:
   - Use chessops `parsePgn` + `startingPosition` + `parseSan`/`makeSanAndPlay` to replay moves
   - After each move, generate a normalized FEN via `makeFen` and strip the last 2 fields (halfmove clock + fullmove number)
   - Check the map for a match, tracking the deepest match found
   - Return the deepest match (most specific opening name)

## Key chessops imports (already used in project)

- `parseFen`, `makeFen` from `chessops/fen`
- `parsePgn`, `startingPosition` from `chessops/pgn`

## Acceptance Criteria

- `classifyOpening('1. e4 e5 2. Nf3 Nc6 3. Bb5')` returns `{ eco: 'C60', name: 'Ruy Lopez' }` (or the appropriate specific variation name)
- `classifyOpening('1. d4 d5 2. c4')` returns `{ eco: 'D06', name: "Queen's Gambit" }` (or appropriate variation)
- Games with no recognizable opening moves return `undefined`
- Build passes, typecheck passes
