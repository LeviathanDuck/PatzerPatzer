# CCP-392-F1 — Add Time Management Profile Section to Stats Page

## Context

CCP-392 was linked to the time management profile task. Audit evidence shows that clock data is captured in `GameSummary` (`hasClockData`, `avgTimePerMove`, `timeTroubleMoves` fields from `src/stats/types.ts`) but no time management stats view exists in `src/stats/view.ts`.

## What This Prompt Must Build

In `src/stats/view.ts`, add a time management profile section per sprint Task 14:

**Content** (only shown when 10+ games have `hasClockData: true`):
- Average time per move (mean of `avgTimePerMove` across games with clock data)
- Time-trouble frequency: percentage of analyzed games where `timeTroubleMoves > 0`
- Blunder-in-time-trouble correlation: compare blunder rate in time-trouble games vs. non-time-trouble games
  - "X% of your blunders happen in time trouble" — compute: sum of blunders from time-trouble games / total blunders
- Simple visual element — a short sentence or small indicator showing the correlation clearly

**Show/hide rule**:
- Section is completely hidden (no container, no placeholder) when fewer than 10 `GameSummary` records have `hasClockData: true`
- When shown, always display how many games the data is based on

**Rules**:
- Snabbdom `h()` only
- Touch only `src/stats/view.ts`
- No chart library
- No new design system

## Validation

- Time management section renders when 10+ games have clock data
- Average time per move displayed
- Time-trouble frequency percentage displayed
- "X% of blunders in time trouble" stat displayed
- Section completely absent (no DOM element) when fewer than 10 clock-data games
- Sample size (number of games with clock data) shown
