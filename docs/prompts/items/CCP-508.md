# CCP-508 — Fix time control classification thresholds in timeClassFromTimeControl

## Context

In `src/import/types.ts`, the `timeClassFromTimeControl` function parses a PGN
`TimeControl` header value (e.g. `"300+0"`) and returns a time class string.

The current thresholds are wrong:

```typescript
if (secs < 180)    return 'ultrabullet';  // treats 1+0, 2+1 as ultrabullet
if (secs < 600)    return 'bullet';       // treats 3+0, 5+0 (blitz) as bullet
if (secs < 1800)   return 'blitz';        // treats 10+0, 15+0 (rapid) as blitz
```

This causes blitz games to show up in the bullet column on the opponent research
library cards, and actual bullet games (1+0, 2+1) to vanish entirely because they
are classified as `'ultrabullet'`, which has no display column.

## Correct Lichess thresholds (based on base time only)

| Class       | Base time |
|-------------|-----------|
| ultrabullet | < 30s     |
| bullet      | < 180s    |
| blitz       | < 480s    |
| rapid       | < 1500s   |
| classical   | >= 1500s  |

Note: Lichess uses estimated total time (base + 40 × increment) for its canonical
classification, but a simple base-time approximation is sufficient here.

## Task

Fix `timeClassFromTimeControl` in `src/import/types.ts` to use the correct
thresholds:

```typescript
if (secs < 30)     return 'ultrabullet';
if (secs < 180)    return 'bullet';
if (secs < 480)    return 'blitz';
if (secs < 1500)   return 'rapid';
return 'classical';
```

Touch only `src/import/types.ts`. No other changes needed.
