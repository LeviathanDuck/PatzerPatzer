# CCP-651 — Polish sprint detail action panels for responsive use

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

Codex has added functional sprint detail action surfaces:
- sprint audit copy panel
- next-phase prompt creation panel
- normalization warning support

Those surfaces need responsive polish so they stay usable on narrower widths and do not become
crowded or awkward inside the sprint detail overlay.

## Relevant file

- `scripts/generate-prompt-dashboard.mjs`

Generated output to inspect:
- `docs/prompts/dashboard.html`

## Goal

Improve the responsive and interaction behavior of the new sprint-detail action surfaces.

Focus on:
- button sizing and wrapping
- code/pre block readability on narrower widths
- panel spacing rhythm in the detail overlay
- keeping action surfaces usable without redesigning sprint semantics

## Hard constraints

- Do not change the backend state model
- Do not remove the always-visible sprint audit copy panel
- Do not change panel visibility rules
- Keep this scoped to responsive/accessibility polish for the new sprint action surfaces

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html`, narrow the window, open sprint detail, and verify:
- action panels remain readable at narrower widths
- copy buttons remain easy to use
- preformatted content does not become unusably cramped
- the prompt and sprint detail views do not regress
