## Context

Manager prompt for the retro live eval diff feature.

This feature makes the Learn From Your Mistakes puzzle session evaluate every move
the user plays during solving against the live engine's best move in real time.
It also notifies the user immediately when the engine determines the original game
move was actually best.

## Child prompts — run in order

1. **CCP-632** — retroCtrl.ts: track live best move, detect game-move vindication
2. **CCP-633** — retroMoveHandler.ts + main.ts: capture solving move, compute dual diffs
3. **CCP-634** — retroView.ts: vindication notice + stacked eval diff boxes
4. **CCP-635** — main.scss: color-coded box styles

Each child prompt must be fully complete and building cleanly before the next begins.

## Feature summary

**Live engine tracking:**
- The live engine continuously updates the best move for the active candidate position.
- Whichever depth is higher — live or batch — wins for both the best move reference
  and the eval value.
- The puzzle's correct answer updates dynamically as the engine deepens.

**Vindication (find state):**
- The moment the engine determines the original game move was actually the best move
  available, a notice appears immediately:
  "Actually upon deeper review, you did play the best move during the game."
- No move needs to be played to trigger this.
- User must manually click Next to advance.

**Dual eval diff boxes (all post-move states):**
- Shown after any move is played: correct, near-best, or wrong.
- Two stacked boxes, always mover-perspective pawn units:
  - Top: solving move vs engine best
  - Bottom: solving move vs original game move

**Color tiers (top box — vs engine best):**
- Green + ✓ — exact best move (diff = 0)
- Yellow — slightly worse (0 to -0.5 pawns from best)
- Blue — better than game move but more than -0.5 from best
- Red — worse than or equal to the original game move

**Color tiers (bottom box — vs game move):**
- Blue — better than game move
- Yellow — equal to game move
- Red — worse than game move

**Sign convention:** always mover-perspective. Positive = better for the player
being reviewed, regardless of color.
