# CCP-643 — Audit sprint dashboard UI and land a design plan

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The sprint tracking system is implemented and the `Sprints` tab is now present in the shared
dashboard, but the visual design was never intentionally polished. The user explicitly wanted the
design review and UI shaping offloaded to Claude, and that step was skipped.

Before broader UI polish prompts are executed, this sprint-tab surface needs a design audit that
identifies the biggest visual problems and establishes a concrete direction for the follow-up
passes.

## Relevant files

- `scripts/generate-prompt-dashboard.mjs`
- generated output to inspect after changes:
  - `docs/prompts/dashboard.html`
- sprint tracking docs for semantic context only if needed:
  - `docs/mini-sprints/SPRINT_REGISTRY_README.md`

## Goal

Do a focused sprint-tab UI audit, identify the main visual problems, and make the smallest safe
structural or styling adjustment needed to support the later polish prompts.

Expected output in Claude's report:
- exact current visual problems
- design direction for the sprint tab
- what was changed now vs what should be left for later prompts

## Scope

This is not the full redesign prompt.

This prompt should:
- inspect the current `Sprints` tab as rendered
- identify the most obvious hierarchy, spacing, density, and scanability issues
- land one small foundational improvement if needed to support the rest of the UI polish work

This prompt should not:
- try to solve every sprint-tab design issue at once
- rewrite the whole dashboard renderer
- change sprint data semantics

## Implementation guidance

Prefer one safe foundational improvement such as:
- improving the sprint-tab container rhythm
- strengthening the overall section framing around the sprint grid and toolbar
- clarifying the visual shell that the later prompts will build on

Keep the actual implementation small and safe, but make the diagnosis strong and useful.

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html` and verify:
- the `Sprints` tab still renders correctly
- the foundational visual cleanup is visible
- the follow-up polish prompts still have clean, well-bounded seams to work on

```sh

```
