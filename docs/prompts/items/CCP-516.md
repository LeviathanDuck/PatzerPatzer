# CCP-516: Wire Opening Classification Into Game Import Paths

## Goal

Modify each game import adapter so that if `opening` or `eco` is missing after PGN header parsing, `classifyOpening()` from `src/openings/eco.ts` fills the gap automatically.

## Depends On

CCP-515 (ECO classifier module must exist)

## Implementation

For each import file, after the existing PGN header extraction of `opening` and `eco`, add:

```typescript
import { classifyOpening } from '../openings/eco';

// ... after extracting opening/eco from headers:
if (!opening || !eco) {
  const classified = classifyOpening(pgn);
  if (classified) {
    if (!opening) opening = classified.name;
    if (!eco) eco = classified.eco;
  }
}
```

### Files to modify

1. **`src/import/pgn.ts`** — after line 23 where `eco` is extracted. This is the highest-value path since pasted PGNs rarely have Opening/ECO headers.

2. **`src/import/lichess.ts`** — after line 50 where `eco` is extracted. Lichess PGNs usually include these headers, but the classifier serves as a fallback.

3. **`src/import/chesscom.ts`** — after line 104 where `eco` is extracted. Chess.com sometimes has incomplete opening names.

## Files

- `src/import/pgn.ts` (modify)
- `src/import/lichess.ts` (modify)
- `src/import/chesscom.ts` (modify)

## Acceptance Criteria

- Import a PGN via paste that has no `[Opening]` or `[ECO]` headers → game gets classified opening name and ECO code
- Import games via Lichess username → all games have opening names (header or classified)
- Import games via Chess.com username → all games have opening names (header or classified)
- Build passes, typecheck passes
