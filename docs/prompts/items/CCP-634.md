### Stacked eval diff boxes (win / fail / view states)

Replace `renderEvalDiff(cand?.evalDiff)` in all three states with a call to
`renderDualEvalBoxes(retro)`.

```typescript
function renderDualEvalBoxes(retro: RetroCtrl): VNode | null {
  const snapshot = retro.getSolvingMoveSnapshot();
  if (!snapshot) return null;

  // --- Box 1: solving move vs engine best ---
  // diff = mover(solvingMove) - mover(engineBest)
  // 0 = exact best → green ✓
  // -0.01 to -0.5 = slightly worse → yellow
  // < -0.5 but better than game move → blue
  // worse than or equal to game move → red
  const vsBestDiff = computeMoverCpDiff(snapshot.solvingMoveCp, snapshot.engineBestCp, snapshot.playerColor);
  const vsGameDiff = computeMoverCpDiff(snapshot.solvingMoveCp, snapshot.gameMoveCp, snapshot.playerColor);

  return h('div.retro-eval-boxes', [
    renderEvalBox(vsBestDiff, 'vs-best', 'vs Engine Best'),
    renderEvalBox(vsGameDiff, 'vs-game', 'vs Move Played'),
  ]);
}
```

**`computeMoverCpDiff(a, b, color)`**: returns `(moverCp(a) - moverCp(b)) / 100` in
pawn units. `moverCp = color === 'white' ? cp : -cp`. Returns `null` if either cp
is undefined.

**`renderEvalBox(diffPawns, key, label)`**:

Tier logic for `vs-best` box:
- `diffPawns === 0` → `grade: 'best'` → display `✓`
- `diffPawns > -0.5 && diffPawns < 0` → `grade: 'near'` → display diff as `-0.3`
- `diffPawns <= -0.5` and solving move better than game → `grade: 'ok'`
- solving move worse than or equal to game move → `grade: 'bad'`

For `vs-game` box:
- `diffPawns > 0` → `grade: 'ok'` (blue — better than game)
- `diffPawns === 0` → `grade: 'near'` (yellow — equal to game)
- `diffPawns < 0` → `grade: 'bad'` (red — worse than game)

```typescript
h('div.retro-eval-box', { class: { [`retro-eval-box--${grade}`]: true } }, [
  h('span.retro-eval-box__label', label),
  h('span.retro-eval-box__value', display),
])
```
