# CCP-664 — PGN file upload for openings import

## Sprint Linkage

- Sprint: SPR-016 (Opponent Research Completion)
- Phase: SPR-016-P2 (Import and Data Integrity)
- Task: SPR-016-T02 (PGN file upload)

## Context

The Opponents page import workflow currently only supports PGN paste via a textarea in
`src/openings/view.ts`. Users need the ability to import PGN files directly via file
browse or drag-and-drop, feeding the parsed file contents into the same existing PGN
import pipeline.

## Task

Add a file input element to the openings import workflow that accepts `.pgn` files via
browse or drag-and-drop. When a file is selected, read its text contents and feed them
into the existing PGN import pipeline — the same path that the paste textarea uses.

## Scope

**Files**: `src/openings/view.ts` (1 file)

- Add an `<input type="file" accept=".pgn">` element near the existing paste textarea
- Support both click-to-browse and drag-and-drop onto the file input area
- Read the selected file using the FileReader API (or equivalent)
- Feed the parsed text into the existing PGN import pipeline
- Do NOT change the import pipeline itself — only add the new input path
- Do NOT touch files outside `src/openings/view.ts`

## Lichess Reference

Check the Lichess PGN import UI for behavioral reference:
- `~/Development/lichess-source/lila/ui/analyse/src/study/studyForm.ts` — study PGN import form
- Adapt the file input pattern to fit the existing openings import workflow
