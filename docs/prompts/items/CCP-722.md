# CCP-722 — Remove Show Engine Button from LFYM Retro Panel and Unify to Main Toggle

## Summary

LFYM engine visibility is split between two independent controls: the main Stockfish toggle in
the engine header (controlled by `retroVisibleEngineEnabled` in `src/ceval/view.ts`) and a
"Show engine" button in the retro panel (controlled by `_guidanceRevealed` in
`src/analyse/retroCtrl.ts`). These do not share state, so the user must press both to fully
reveal engine output. This prompt removes the duplicate control.

## Task

Remove the "Show engine" button from the LFYM retro panel and unify all engine visibility
to the main Stockfish toggle.

### Required files to inspect first

- `src/analyse/retroView.ts` — lines 597-604: "Show engine" button rendering
- `src/analyse/retroCtrl.ts` — `_guidanceRevealed`, `revealGuidance()`, `guidanceRevealed()`
- `src/ceval/view.ts` — lines 65-76: `retroVisibleEngineEnabled`, lines 133-137: main toggle
  click handler during LFYM
- `src/engine/ctrl.ts` — line 288: `retroHidden` gate uses `guidanceRevealed()`
- `src/main.ts` — lines 929-963: PV/eval bar rendering gates using both flags

### Implementation

1. In `src/analyse/retroView.ts`:
   - Remove the "Show engine" button block (lines ~597-604)
   - Remove the `showEngineBtn` variable and its insertion into the feedback area
   - Remove the `onRevealGuidance` callback from the function signature if it's no longer needed

2. In `src/engine/ctrl.ts`:
   - Change the `retroHidden` gate (line 288) from:
     ```typescript
     const retroHidden = ctrl.retro !== undefined && !ctrl.retro.guidanceRevealed();
     ```
     to use the `retroVisibleEngineEnabled` flag from `src/ceval/view.ts`:
     ```typescript
     const retroHidden = ctrl.retro !== undefined && !isRetroVisibleEngineEnabled();
     ```
   - Import `isRetroVisibleEngineEnabled` from `src/ceval/view.ts`

3. In `src/main.ts`:
   - Remove the `onRevealGuidance` callback passed to `renderRetroBox()` if it's no longer used
   - Simplify the PV line gate (`showRetroPv`) to use only `retroVisibleEngineEnabled`:
     ```typescript
     const showRetroPv = !ctrl.retro || retroVisibleEngineEnabled;
     ```
   - The eval bar already uses `retroVisibleEngineEnabled` — no change needed there

### Constraints

- Touch only `src/analyse/retroView.ts`, `src/engine/ctrl.ts`, and `src/main.ts`
- Do NOT remove `_guidanceRevealed` / `guidanceRevealed()` / `revealGuidance()` from retroCtrl
  yet — they may still be needed by other code paths; cleanup is a follow-up if needed
- The main Stockfish toggle click handler in `src/ceval/view.ts` (lines 133-137) already works
  correctly for LFYM — do not change it
- Engine arrows, PV lines, eval bar, and pearl must all gate on the same unified flag

## Validation

- The "Show engine" button no longer appears in the LFYM retro panel
- Clicking the main Stockfish toggle during LFYM reveals all engine output (arrows, PV lines,
  eval bar, pearl)
- Clicking the main Stockfish toggle again hides all engine output
- No double-click required — one toggle controls everything
- Arrow shapes update immediately when the toggle is clicked
- npm run build passes
- npx tsc --noEmit passes
