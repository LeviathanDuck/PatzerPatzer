# CCP-518: Add IDB Eco Index And Opening Filter In Games Tab

## Goal

Add IndexedDB indexes on opening/eco fields for future query optimization, and add an opening filter to the Games tab UI.

## Depends On

CCP-517 (backfill must have run so games have opening data to filter on)

## Implementation

### 1. IDB schema upgrade in `src/idb/index.ts`

- Bump `DB_VERSION` (check current version, increment by 1)
- In the `onupgradeneeded` handler, add a version check for the new version:
  - Open existing `games` store via `tx.objectStore('games')`
  - Add indexes: `eco` and `opening` (both non-unique)
- IndexedDB builds the indexes automatically over existing data

### 2. Opening filter in `src/games/view.ts`

- Add a text input filter for opening name/ECO code alongside existing filters
- Filter the displayed games list by case-insensitive substring match against `game.eco` or `game.opening`
- This is a client-side filter on the already-loaded games array (the IDB index enables future cursor-based optimization but is not needed for v1 filtering)

## Files

- `src/idb/index.ts` (modify — version bump + indexes)
- `src/games/view.ts` (modify — filter UI)

## Acceptance Criteria

- App loads without IDB errors (upgrade handler runs cleanly)
- Games tab shows an opening filter input
- Typing "Sicilian" filters to only games with Sicilian in the opening name
- Typing "B20" filters to games with that ECO code
- Filter is case-insensitive
- Build passes, typecheck passes
