# CCP-655 — Restore "Opponents" label on non-target color toggle button

## Context

On the Opponents opening tree page, the color toggle (`openings__color-toggle`) has
two buttons — White and Black. Each button has an `openings__color-username` span
above the color label:

- The active/selected button shows the imported player's username (e.g. "marox")
- The inactive button should show "Opponents" in the same span

The "Opponents" text was accidentally removed — the non-active button's
`openings__color-username` span was set to empty string `''` instead of
`'Opponents'`. These labels should also swap when the active color changes.

## Location

`src/openings/view.ts` — `renderColorToggle` function (~line 1988).

Two occurrences:
- White button: `filter === 'white' ? playerName : ''` → `filter === 'white' ? playerName : 'Opponents'`
- Black button: `filter === 'black' ? playerName : ''` → `filter === 'black' ? playerName : 'Opponents'`

## Validation

- [ ] Black button shows username when Black is selected; "Opponents" when White is selected
- [ ] Labels use the same `openings__color-username` span and styling as the username
- [ ] Build passes cleanly
