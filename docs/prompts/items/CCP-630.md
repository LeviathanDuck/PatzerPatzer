## Context

CCP-629 replaced the three severity pills with a 3-step discrete slider. This prompt
replaces that with a fully continuous slider so the user can set any loss threshold
from 1% to 25% in 1% steps.

Divider markers on the gradient track show where each classification label begins:
- Inaccuracy: loss ≥ 5% (20% along the 1–25 track)
- Mistake: loss ≥ 10% (40% along the track) — also the Lichess default
- Blunder: loss ≥ 15% (60% along the track)

The data model must change: `minClassification: 'inaccuracy' | 'mistake' | 'blunder'`
is replaced by `minLossThreshold: number` (stored as a fraction, e.g. 0.10 for 10%).
`retro.ts` uses the threshold directly instead of mapping from a classification enum.

## Target files

- `src/analyse/retroConfig.ts` — replace `minClassification` with `minLossThreshold`
- `src/analyse/retro.ts` — use `retroConfig.minLossThreshold` directly
- `src/header/index.ts` — update slider to continuous range, update value label, remove classification label references
- `src/styles/main.scss` — add divider marker styles, remove no-longer-needed 3-step styles if any

Note: 4 files is unavoidable here — the data model change and its consumer must move
together, and the UI and its styles are inseparable.

```ts
/**
 * Minimum win-chance loss to include a move as a candidate. Range [0.01, 0.25].
 * Stored as a fraction (0.10 = 10%). Default: 0.10 (Mistake threshold, Patzer default).
 * Lichess parity (inaccuracy floor): 0.05. Blunder threshold: 0.15.

minLossThreshold: number;
```
