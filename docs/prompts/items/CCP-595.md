# Eval diff for retro move quality — batch manager

## Purpose

Implement the data plumbing and display infrastructure to show the evaluation
difference between the engine's best move and the move the user actually played,
surfaced in the "Learn From Your Mistakes" interface.

The goal: when a user solves (or fails) a retro exercise, the UI can show
something like "the best move gains +2.0 over what was played in the game."
The exact copy/wording is deferred — this sprint builds the backend correctly
so messaging can be layered on top without re-architecting.

## Design principles

- **Engine in secret**: background eval requests for best-move positions that
  are not yet in the eval cache run silently, without affecting the analysis
  board, eval graph, or any visible engine output.
- **Reusable from day one**: the eval-diff utility lives in a shared module
  (`src/analyse/evalDiff.ts`) so puzzles and other tools can import it later
  without duplication.
- **No messaging decisions yet**: CCP-598 adds a placeholder value display.
  The exact copy ("X better than what was played", "You gained X", etc.) is
  a separate future polish task.

## Child prompts — run in order

1. **CCP-596** — Create `src/analyse/evalDiff.ts`: shared pure utility for
   computing and formatting the eval difference between two positions.

2. **CCP-597** — Extend `RetroCandidate` with an `evalDiff` field; populate
   it at session-build time from the existing eval cache; wire a silent
   background engine evaluation pass for candidates where best-move position
   data is missing or shallow.

3. **CCP-598** — Read `cand.evalDiff` in `src/analyse/retroView.ts` and
   render a placeholder diff value in the feedback panel and solution reveal.

## Out of scope for this sprint

- Exact display copy ("X better than what was played")
- Puzzle interface integration (transfer to puzzles is a separate later task)
- EvalDiff display in the move list or eval graph
- Per-move diff history across the whole game
