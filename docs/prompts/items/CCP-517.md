# CCP-517: Backfill Existing Games And Wire Openings Research Import

## Goal

Classify existing games in IDB that lack opening/ECO data, and wire the classifier into the openings research import path.

## Depends On

CCP-515 (ECO classifier module), CCP-516 (import wiring pattern)

## Implementation

### 1. Backfill function in `src/idb/index.ts`

Add `backfillOpenings()` async function:
- Open a readonly cursor on the `games` store
- Collect records where `opening === null || eco === null`
- For each, call `classifyOpening(record.pgn)` from `src/openings/eco.ts`
- Write updated records back in a readwrite transaction
- Return the count of updated records
- Log count to console for observability

### 2. Wire into openings research import

In `src/openings/import.ts`, add the `classifyOpening` fallback to `pgnToResearchGame()` (after line 56-57 where opening/eco are extracted from headers). Same pattern as CCP-516.

### 3. Trigger at startup

In `src/main.ts`, call `backfillOpenings()` after `loadGamesFromIdb()` resolves. Fire-and-forget — runs in background, does not block UI.

## Files

- `src/idb/index.ts` (modify — add `backfillOpenings()`)
- `src/openings/import.ts` (modify — wire classifier into `pgnToResearchGame`)
- `src/main.ts` (modify — one-line startup call)

## Acceptance Criteria

- Existing games that previously showed "–" for opening now show classified names after app reload
- Console logs "Backfilled opening data for N games" (or similar)
- Research game imports also get classified openings when PGN headers are missing
- Build passes, typecheck passes
