Source step: prompt dashboard bulk-copy formatting bug fix

## Context

The prompt dashboard header includes two bulk-copy buttons:
- `Copy Needs Review IDs`
- `Copy Issues Found IDs`

Current bug:
- the copied output includes numbered-list prefixes such as `1. CCP-323, 2. CCP-355`
- the user wants plain comma-separated prompt IDs only, with no extra numbering

Expected behavior:
- copied output should be raw comma-separated prompt IDs such as:
  - `CCP-323, CCP-355, CCP-363`
- no leading ordinal numbers should be added
- both bulk-copy buttons should use the same plain-ID format

## Target files

- `scripts/generate-prompt-dashboard.mjs`
- generated output: `docs/prompts/dashboard.html`

Do not hand-edit `docs/prompts/dashboard.html`.

## Required workflow

1. Inspect the current bulk-copy button logic in `scripts/generate-prompt-dashboard.mjs`.
2. Confirm where the numbering is added.
3. Apply the smallest safe fix so the copied values are plain comma-separated IDs only.
4. Keep the button behavior, filters, and feedback states unchanged outside this format fix.
5. Regenerate the dashboard after the change.

## Implementation guidance

- keep this fix scoped to the copied string format only
- do not redesign the buttons or the header layout
- preserve:
  - which prompts each button targets
  - the `None` fallback behavior
  - the `✓ Copied!` success feedback
- prefer changing the ID mapping at the source rather than post-processing a numbered string
