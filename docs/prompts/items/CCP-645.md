# CCP-645 — Refine sprint detail panel information hierarchy

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The sprint detail panel currently exposes the right data, but the presentation is crowded and
flat. Dependencies, audits, recommended next steps, phases, tasks, and unassigned prompts all
compete for attention in similar visual containers.

This is making the detail view harder to read than it needs to be, especially for large sprints
with many phases and tasks.

## Relevant files

- `scripts/generate-prompt-dashboard.mjs`
- generated output to inspect after changes:
  - `docs/prompts/dashboard.html`

## Goal

Improve the sprint detail overlay so the information hierarchy is clearer and the user can more
quickly distinguish:
- summary/meta information
- dependencies and audits
- recommended next steps
- phases and tasks
- unassigned prompts

## Constraints

- Keep this scoped to sprint detail-panel structure and styling
- Do not change sprint data semantics
- Do not redesign the full dashboard shell
- Do not edit generated `docs/prompts/dashboard.html` directly

## Design direction

Use grouping and hierarchy, not just smaller text tweaks. The detail view should feel like a
purposeful inspection panel instead of one long list of similarly styled blocks.

Good improvements include:
- clearer section separation
- stronger task-row hierarchy
- better treatment for recommended next steps so they read as actionable
- more readable phase framing for large sprints

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html`, open several sprint details, and verify:
- the detail overlay is easier to scan
- recommended next steps feel distinct from audits and dependencies
- phases and task rows are easier to parse in large sprints
- unassigned prompts still remain visible without overwhelming the rest of the panel
