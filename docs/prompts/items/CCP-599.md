Fix 3 bugs found during review of the Study Page sprint (CCP-522–CCP-563):

---

**Bug 1 (Data loss — CCP-541):** `persistStudy()` in `src/study/studyDetailCtrl.ts` does not regenerate the PGN from the tree before saving. All annotations (comments, glyphs, shapes, variations) are lost on reload because the original PGN is written back unchanged.

Fix: In `persistStudy()`, set `pgn: buildStudyPgn()` on the updated StudyItem before calling `saveStudy()`.

File: `src/study/studyDetailCtrl.ts` — the `persistStudy` async function (~line 155).

---

**Bug 2 (Layout regression — CCP-540):** `move-wrap` custom element is styled as `display: inline-flex` in `src/styles/main.scss`. This makes it a flex child of `.tview2.tview2-column` instead of `move`, breaking the 13% / 43.5% / 43.5% column grid alignment.

Fix: Change `move-wrap { display: inline-flex }` to `move-wrap { display: contents }`. This makes move-wrap transparent to the flex layout, so `move` and `button.bookmark-btn` appear as direct flex children — which is exactly what we want.

File: `src/styles/main.scss` — the `move-wrap` rule (~line 2727).

---

**Bug 3 (Dead code — CCP-532):** `src/study/libraryView.ts` contains a stub `renderStudyDetail()` function (~line 382) that was left over from Phase 0. The real implementation is in `src/study/studyDetailView.ts`. The stub returns a "coming soon" placeholder and is never called (main.ts imports from studyDetailView), but it's confusing dead code.

Fix: Remove the stub `renderStudyDetail` function and its export from `libraryView.ts`.
