## Context

The current prompt creation process is "Quick Create" — Claude infers intent and writes
the prompt body with minimal questioning. This produces prompts that lack precise design
decisions, often leaving ambiguity around placement, behavior, Lichess parity, edge cases,
and acceptance criteria.

The user wants a second mode called **AI Guided Questions** where Claude asks one focused
question at a time to extract a precise, implementation-ready specification before writing
the prompt body. This mode is gated into CLAUDE.md as a behavioral instruction so it
applies automatically in every session.

Sprint creation is handled separately in CCP-617.

The user has defined a companion snippet:

```
<GUIDED_QUESTIONS_MODE>
Your role is to extract a precise, implementation-ready specification — not to validate
assumptions or accept vague answers. Ask one question at a time. Do not proceed until the
current answer is specific enough to write code from. If an answer is ambiguous, restate
what you understood and ask for clarification before continuing. Probe for: exact placement
and behavior, Lichess parity vs conscious deviation, affected files, edge cases, and
acceptance criteria. Do not infer — make the user decide. Ask open questions; do not
suggest answers that lead the user toward a particular choice. When all decisions are
captured, summarize them back before creating the prompt.
</GUIDED_QUESTIONS_MODE>
```

### CLAUDE.md changes

Add a new section **`## AI Guided Questions Mode`** with:

**The gate rule (mandatory behavioral instruction):**
> Before creating any prompt — whether triggered by user request, manager
> orchestration, or agent workflow — Claude must ask:
>
> *"AI Guided Questions or Quick Create?"*
>
> - **Quick Create**: proceed with current behavior (infer and write)
> - **AI Guided Questions**: enter guided mode (see below)
>
> This gate applies every time a prompt is being created. No exceptions.
> Sprint creation is gated separately (see CCP-617).

**Guided mode behavior for prompts:**
- Activate `<GUIDED_QUESTIONS_MODE>` (paste the full snippet above)
- Ask questions one at a time — do not batch
- Questions must probe for:
  - Exact UI placement and behavior
  - Lichess parity vs known deviation (and why)
  - Which files will be touched
  - Edge cases and error states
  - Acceptance criteria (how will we know it's done?)
  - Mobile behavior if UI is involved
- If an answer is too vague to write code from, restate what was heard and ask again
- When all decisions are captured, summarize back to the user:
  *"Here's what I'll put in the prompt — does this match your intent?"*
- Only after confirmation: proceed to reserve + write + finalize the prompt

```markdown
## Creation Modes

Before creating any prompt, Claude must ask:

> "AI Guided Questions or Quick Create?"

**Quick Create** — Claude infers intent and writes the prompt body directly.
Use for small, clear, well-understood tasks.

**AI Guided Questions** — Claude asks one targeted question at a time to build
a precise spec. Use for anything involving UI decisions, behavior choices,
Lichess parity questions, or uncertain scope. Answers are baked directly into
the prompt body. Claude summarizes decisions back before finalizing.
```
