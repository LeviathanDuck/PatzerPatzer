# Render eval diff placeholder in retroView feedback and solution panels

## Context

Part of CCP-595 (eval diff for retro move quality). This is child prompt 3/3.
Depends on CCP-596 and CCP-597 being complete (`evalDiff` field populated on
`RetroCandidate`, `EvalDiff` type and `formatPawns` available).

The data is wired. Now surface it in the UI — as a placeholder that shows the
numeric value. Final copy/messaging ("That move gains +2.0 over what was played
in the game", etc.) is a separate future polish task.

## File to touch (1)

- `src/analyse/retroView.ts`

## Where to display

### 1. Win feedback panel (feedback === 'win')

After the existing "Best move" or "Well done" text, append the eval diff if
`cand.evalDiff` is available. Example placeholder:

```
[existing win text]
Diff: +2.0
```

Only show if `cand.evalDiff` exists. If the diff is 0.0 or negative (played
move matched or exceeded best), show nothing — do not show "Diff: 0.0".

### 2. Fail feedback panel (feedback === 'fail')

After the existing "Better, but not the best move" / "That move is even worse"
text, show the diff. Example:

```
[existing fail text]
Diff: +1.5 vs your move
```

Again, only when `cand.evalDiff` is populated.

### 3. Solution reveal panel (feedback === 'view')

After the solution move SAN, show the diff. Example:

```
Best was Nf3
Diff: +2.0
```

Only when `cand.evalDiff` is populated.

## Implementation notes

- Import `formatPawns` from `'./evalDiff'`.
- The diff value to display is `cand.evalDiff.formatted` — already formatted
  by the utility (e.g., "+2.0").
- Wrap in a `h('span.retro-feedback__eval-diff', ...)` or similar minimal
  Snabbdom element so it can be styled separately later.
- Do NOT add any CSS rules yet — the element will inherit existing text styles.
  A dedicated styling task will follow.
- Gate every reference with `cand.evalDiff != null` — never assume it's present.

## Example Snabbdom pattern

```typescript
// In the appropriate feedback section:
...(cand.evalDiff
  ? [h('span.retro-feedback__eval-diff', `Diff: ${cand.evalDiff.formatted}`)]
  : []),
```

## What this does NOT do

- Does not change any logic or data — display only
- Does not add CSS or styling
- Does not change copy/wording of existing feedback text
- Does not display diff for candidates that haven't been resolved yet (find state)

## Validation

- `npm run build` passes
- `npx tsc --noEmit` passes
- Open "Learn from your mistakes" on a fully reviewed game
- Solve an exercise correctly → win panel shows "Diff: +X.X" (if diff available)
- Fail an exercise → fail panel shows "Diff: +X.X" (if diff available)
- Click "Show solution" → solution panel shows "Diff: +X.X" (if diff available)
- Candidates with no eval diff available → no "Diff:" text shown at all
