Fix 2 bugs found during line-by-line review of the annotation panel work (CCP-534):

---

**Bug 1 (Comment display — annotationView.ts line 24):** `renderCommentPanel()` reads
`node?.comments?.[0]?.text` to show the current comment, but `saveComment()` finds
and updates by `c.id === 'user'`. If a node has a PGN comment at index 0 (different id)
followed by a user comment at index 1, the wrong text is displayed and pre-filled in
the editor.

Fix: Change line 24 from:
```typescript
const currentComment = node?.comments?.[0]?.text ?? '';
```
to:
```typescript
const currentComment = node?.comments?.find(c => c.id === 'user')?.text ?? '';
```

---

**Bug 2 (Cancel saves instead of discards — annotationView.ts line 44):** The textarea's
`blur` event fires BEFORE the Cancel button's `click` event. So clicking Cancel triggers
`saveComment()` via blur first, persisting the draft — Cancel does nothing useful.

Fix: Change the Cancel button from `click` to `mousedown` with `e.preventDefault()`:

```typescript
h('button.study-btn', {
  on: {
    mousedown: (e: MouseEvent) => {
      e.preventDefault(); // prevents textarea from blurring, suppresses blur→saveComment
      cancelCommentEdit();
      redraw();
    },
  },
}, 'Cancel')
```

`e.preventDefault()` on `mousedown` prevents focus transfer away from the textarea,
so the textarea's `blur` handler never fires.
