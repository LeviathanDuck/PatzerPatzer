# CCP-648 — Polish sprint audit copy panel design

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

Codex has already implemented the sprint-detail backend contract for sprint audit actions.

The sprint detail view now always includes a functional sprint-audit copy panel backed by
precomputed data:
- `auditPromptTemplate`
- sprint status and phase/task state rollups
- normalization readiness
- latest audit references
- linked prompt coverage

This functionality exists, but the visual presentation is still utilitarian. The user wants
Claude to make the sprint audit copy surface feel designed and intentional.

## Relevant file

- `scripts/generate-prompt-dashboard.mjs`

Generated output to inspect:
- `docs/prompts/dashboard.html`

## Goal

Improve the visual design of the always-visible sprint audit copy panel in sprint detail.

Focus on:
- hierarchy and framing
- copy-button treatment
- making the panel feel important but not overwhelming
- making the long audit prompt content feel manageable on a dark dashboard surface

## Hard constraints

- Do not change sprint semantics, registry logic, or audit-template generation logic
- Do not rename or remove the backend fields Codex added
- Do not redesign the whole sprint detail view in this prompt
- Keep this scoped to the sprint audit copy panel presentation and interaction treatment

## Backend contract already implemented

Consume the existing backend fields as-is:
- always-visible audit panel in sprint detail
- `auditPromptTemplate`
- current sprint status
- normalization readiness
- next available phase context

Do not invent new sprint state meanings.

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html`, open several sprint details, and verify:
- the sprint audit copy panel feels visually intentional
- the copy action is obvious and pleasant to use
- long audit prompt content is easier to read or manage
- no regressions appear in prompt detail or the rest of sprint detail
