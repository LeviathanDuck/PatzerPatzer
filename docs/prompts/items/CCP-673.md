# CCP-673 — Unify Sprint Panel Archival To RETIRED-V# Pattern

Prompt ID: CCP-673
Task ID: CCP-673
Source Document: (audit conversation, 2026-04-02)

## Context

Prompt editing archives old versions using the `RETIRED-PROMPT-DONOTRUN-REFERENCE-ONLY-<ID>-V#` pattern
in the prompt registry. Sprint panel editing uses a different approach: a `panelNotes.history` array
on the sprint record. This inconsistency should be unified — sprint panels should use the same
RETIRED-V# archival pattern.

CCP-672 must be completed first (the textarea editor must be in place).

## Task

Migrate sprint panel archival from `panelNotes.history` to the RETIRED-V# pattern used by prompts.

### Current sprint panel archival (to remove)

In `scripts/sprint-registry-lib.mjs`, sprint panel saves store previous versions in:
```
sprint.panelNotes[panel].history = [
  { body: "old version", archivedAt: "2026-04-02T..." },
  ...
]
```

### New sprint panel archival (to implement)

When a sprint panel body is saved:
1. Create a new entry in the sprint registry with a retired reference ID pattern:
   `RETIRED-SPRINT-PANEL-DONOTRUN-REFERENCE-ONLY-<SPRINT_ID>-<PANEL>-V#`
2. The retired entry stores:
   - `sprintId` — which sprint it belongs to
   - `panel` — which panel (audit, mismatch, nextPhase, appendRequest)
   - `archivedBody` — the previous panel body
   - `archivedAt` — timestamp
   - `supersededFromSprintId` — the originating sprint ID
   - `retiredReferenceOnly: true`
   - `hiddenFromDashboard: true`
3. The active sprint `panelNotes[panel]` keeps:
   - `currentBody` — the new saved body
   - `lastEditedAt` — timestamp
   - `supersededVersionIds` — array of retired reference IDs (like prompts do)
4. Remove the `history` array from `panelNotes[panel]`

### Files to touch

1. `scripts/sprint-registry-lib.mjs` — archival logic change
2. `docs/mini-sprints/sprint-registry.json` — will gain retired entries on next save

### Migration

Existing `panelNotes.history` entries do not need to be migrated. They can be left in place
as inert data. New saves going forward will use the RETIRED-V# pattern.

## Validation

1. Start `node server.mjs`
2. Open a sprint in the dashboard
3. Edit a sprint panel body and save
4. Check `sprint-registry.json` — a `RETIRED-SPRINT-PANEL-DONOTRUN-REFERENCE-ONLY-*` entry should exist
5. The active sprint's `panelNotes[panel]` should have `supersededVersionIds` array, no new `history` entries
6. Edit and save the same panel again — V2 should be created
7. Verify retired entries have `hiddenFromDashboard: true` and don't appear in the sprint list
