# CCP-644 — Improve sprint progress and status visualization

Read and follow:
- `/Users/leftcoast/Development/PatzerPatzer/CLAUDE.md`
- `/Users/leftcoast/Development/PatzerPatzer/AGENTS.md`

## Context

The sprint cards and sprint detail view already show useful status data:
- sprint status badge
- plan coverage
- execution progress
- implementation progress
- segmented progress bar by task state

But the current presentation is visually weak. The progress stack reads as flat text, the
relationship between the three percentages is not especially intuitive, and the status treatment
does not yet feel polished enough for a dashboard meant to communicate planning truth,
execution truth, and audit truth.

## Relevant files

- `scripts/generate-prompt-dashboard.mjs`
- generated output to inspect after changes:
  - `docs/prompts/dashboard.html`

## Goal

Make progress and status information read clearly and feel visually deliberate.

Improve:
- the readability of the three progress dimensions
- the visual treatment of the segmented progress bar
- the hierarchy between overall sprint status and supporting progress metrics
- the consistency of status styling between sprint cards and sprint detail

## Constraints

- Keep this scoped to sprint progress/status visualization
- Do not redesign the full card layout beyond what this visualization needs
- Do not change sprint math, registry data, or state names
- Do not edit generated HTML directly

## Design direction

The sprint tab should make it easy to answer:
- how planned-out is this sprint
- how much execution happened
- how much is actually implemented / verified

Use visual hierarchy, labels, spacing, and bar treatment to make those distinctions clear.

## Validation

Run:
```sh
npm run dashboard:generate
```

Open `docs/prompts/dashboard.html` and verify:
- plan coverage, execution progress, and implementation progress are easier to distinguish
- the segmented progress bar feels intentional and legible
- sprint status badges and progress indicators feel consistent between card and detail surfaces
- no prompt-tab visuals regress
