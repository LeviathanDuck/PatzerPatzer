# CCP-660 — Opening Tree Date Range Filter and Speed Filter Fix

## Context

The Opening Tree session view has a speed filter (Time control chips) rendered below the moves
list. Two related improvements are needed:

1. **Speed filter behavior fix** — the current logic is inverted: clicking a chip *removes* that
   speed from the selection. The correct behavior is: default = all speeds active, clicking a chip
   *narrows* to just that speed, additional clicks *add* to the selection, and if all chips end up
   selected again it resets back to the "all" default.

2. **Date range filter** — a new filter row below the speed chips that lets the user filter the
   opening tree by recency. Changing the date range triggers a full tree rebuild (same as the
   speed filter) and the active filter is shown as a badge in every tool panel.

## Task

### Part 1: Fix speed filter click behavior

Current behavior: clicking a chip removes it from the active set (deselect logic).
Required behavior:
- Default state: all speeds active (empty set = all, same shorthand as today)
- Clicking a chip when all are active: narrows to that chip only
- Clicking a chip when some are active: adds it to the active set
- If all chips end up selected: collapse back to the empty "all" shorthand
- Clicking the only active chip: do not leave an empty set — stay on that chip (or snap back to all)

### Part 2: Add date range filter row

**Location**: new row directly below the speed filter chips, inside the same opening tree panel.

**Default state**: no filter active.  
**Default button label**: `Date Range: All (N)` where N = total games in the collection for the
currently selected colour (ignoring speed filter).

**Options** (shown in a small popup on button click, same popup style as other filter dropdowns):
- Last Week
- Last Month
- Last 3 Months
- Last 6 Months

Each option shows a raw game count for that date window, computed from the full collection for the
selected colour — **ignoring** any active speed filter. These are "available games" counts, not
post-filter counts.

**Active state button label**: `{Range} ({N})` e.g. `Last Month (45)`.  
An × button appears inline to clear back to "All".

**Popup behaviour**: opens on button click, dismisses on click-outside or option select.

**Tree rebuild**: selecting or clearing the date range filter must trigger a full tree rebuild
identical to how `setSpeedFilter` works — node counts, percentages, and move statistics must all
reflect only games within the selected date window.

**Scope**: the date range filter affects all tools in the session — Opening Tree, Prep Report,
Style, and Deviation Scan all read from the rebuilt tree.

**Filter badge**: when a non-default date range is active, a small dismissible pill (e.g.
`Last 3 Months ×`) must appear at the top of every tool panel (Opening Tree, Prep Report, Style,
Deviation Scan). Clicking × clears the filter and triggers a tree rebuild back to all-time.
