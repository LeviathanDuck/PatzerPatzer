# CCP-719 — Replace Unicode Color Indicators with CSS Circles in Opening Tree

## Summary

The Opening Tree color toggle in `renderColorToggle()` uses unicode text characters (`○`/`●`)
instead of the CSS circle pattern used by `.player-strip__color-icon` elsewhere in the app.
This makes it harder to read which side the player is on at a glance.

## Task

Replace the unicode color indicators with CSS-styled circles matching the analysis board pattern.

### Required files to inspect first

- `src/openings/view.ts` — `renderColorToggle()` function, uses `.openings__color-dot` with text
- `src/board/index.ts` — `.player-strip__color-icon` reference implementation
- `src/styles/main.scss` — existing `.player-strip__color-icon--white` and
  `.player-strip__color-icon--black` SCSS rules

### Implementation

In `src/openings/view.ts`, in the `renderColorToggle()` function:

1. Replace the `○` unicode character (white indicator) with:
   ```typescript
   h('span.player-strip__color-icon.player-strip__color-icon--white')
   ```

2. Replace the `●` unicode character (black indicator) with:
   ```typescript
   h('span.player-strip__color-icon.player-strip__color-icon--black')
   ```

3. The existing SCSS rules for these classes should produce an 11px diameter circle with
   border-radius 50% and appropriate background/border colors. Verify the SCSS rules exist
   and match this description. If `.openings__color-dot` had any custom sizing or spacing,
   ensure the replacement circles have the same visual footprint.

### Constraints

- Touch only `src/openings/view.ts`
- Reuse the existing `.player-strip__color-icon` classes — do NOT create new CSS classes
- Do NOT change the player strip layout, toggle behavior, or any other rendering
- If `.openings__color-dot` is no longer referenced after this change, leave it in SCSS
  for now (cleanup is a separate task)

## Validation

- White indicator shows as a styled CSS circle (not unicode `○`)
- Black indicator shows as a styled CSS circle (not unicode `●`)
- Circle size and style match the analysis board player strip indicators
- Color toggle still works (clicking switches the active color)
- No layout shifts or spacing changes
- npm run build passes
- npx tsc --noEmit passes
