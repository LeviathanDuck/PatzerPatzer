# CCP-641 — Recommended Next Box: Fallback to Unrun Queued Prompts

## Context

The dashboard's "Recommended Next" box (`renderUrgentBlock` in `scripts/generate-prompt-dashboard.mjs`)
currently shows only prompts that are `queued-pending` AND meet a high-priority filter:
`priority === 'critical' | 'high'` OR `category === 'bugfix' | 'typecheck'`.

When no prompts meet those criteria the box disappears entirely. The user wants it to remain
useful by falling back to the next batch of un-run queued prompts.

## Task

Modify `renderUrgentBlock` in `scripts/generate-prompt-dashboard.mjs` so that:

1. **Primary list** — existing criteria unchanged: `queued-pending` prompts that are
   `critical`/`high` priority or `bugfix`/`typecheck` category, sorted as today.

2. **Fallback list** — when the primary list is empty, show up to **5** `queued-pending`
   prompts sorted ascending by numeric ID (lowest CCP number first = earliest created =
   the natural completion order).

3. **Manager deduplication** — for both lists, before rendering:
   - Build a reverse index: for every manager prompt (`kind === 'manager'`) collect its
     `batchPromptIds`.
   - If a prompt in the candidate list is a member of any manager's `batchPromptIds`,
     replace it with the manager prompt (if not already in the list).
   - Remove the original child prompt from the list.
   - De-duplicate: if the manager is already in the list, don't add it twice.
   - Apply this deduplication **before** the 5-item cap on the fallback list.

4. **Label** — when displaying the fallback list, change the heading count suffix to
   read `"N prompt(s) — queued"` (instead of the primary list's `"N prompt(s)"`) so the
   user can tell which mode is active.  A simple CSS class or inline style change is fine.

## Scope

- **Only file to touch:** `scripts/generate-prompt-dashboard.mjs`
- Do not change `isUrgent`, `renderList`, sort logic, CSS, or any other function.
- The change is entirely within `renderUrgentBlock` and any small helper built inside it.

## Validation

- With critical/high/bugfix/typecheck prompts present: box shows them (unchanged behavior).
- With none of those present: box shows up to 5 queued-pending prompts, lowest IDs first.
- If any of the shown prompts are children of a manager, the manager ID appears instead.
- The fallback heading count says "N prompt(s) — queued".
- `npm run prompts:refresh` succeeds with no errors after the change.
