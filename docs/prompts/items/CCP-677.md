# CCP-677 — Fix Sprint Grid Updated-At Accuracy And Add Created-At Timestamp

Prompt ID: CCP-677
Task ID: CCP-677
Source Document: user request 2026-04-02

## Context

The sprint grid currently shows "updated X minutes ago" on every sprint card. This timestamp
is inaccurate because `sprint.updatedAt` gets stamped on the entire sprint record every time
the registry is mutated — not just when the sprint itself was meaningfully changed. The result
is all sprints showing the same recent timestamp even if they haven't been touched.

Additionally, there is no "created at" timestamp visible on sprint cards, which would help
orient the user when reviewing the grid.

## Task

Fix the sprint grid to show accurate per-sprint timestamps, and add a created-at timestamp
to each card.

### 1. Fix the updated-at timestamp

The root cause is that `sprint.updatedAt` is set to `new Date().toISOString()` at the end
of every `mutateSprintRegistryLocked` call in `server.mjs`, even when the sprint being
mutated is a different sprint (e.g., a RETIRED panel archival entry is being added).

Fix: only stamp `sprint.updatedAt` on the sprint that was actually changed in the mutation,
not on every sprint that goes through the loop.

In `server.mjs` in `handleSprintPanelNoteWrite`, the line:
```js
sprint.updatedAt = new Date().toISOString();
```
is at the end of the mutation callback and applies to the active sprint being saved. That
part is fine. Check that no other code path is globally stamping unrelated sprints.

Also check `scripts/sprint-registry-lib.mjs` — if `recomputeSprintRegistryState` or any
normalization function is touching `updatedAt` on sprints that were not actually changed,
that must be removed.

### 2. Add created-at timestamp to sprint cards

In `scripts/generate-prompt-dashboard.mjs`, in the sprint card rendering function
(search for `sprint-card` CSS class or the function that builds the card HTML):

- Add a `createdAt` display below the `updatedAt` display on each sprint card
- Format: `Created: <date>` using the same `fmtDate` helper already used for other dates
- Style: small, centered, muted text below the existing "Updated X ago" line
- Only render the created-at line if `sprint.createdAt` is present

### Files to touch

1. `scripts/generate-prompt-dashboard.mjs` — add created-at to sprint card
2. `docs/prompts/dashboard.html` — regenerated output (via `npm run dashboard:generate`)
3. `server.mjs` — verify/fix updatedAt stamping scope (if needed)

## Validation

1. Open the sprint grid in the dashboard (served via `node server.mjs`)
2. Verify each sprint card shows a "Created: <date>" below the updated timestamp
3. Edit a sprint panel body and save — only that sprint's updatedAt should change
4. Other sprints should still show their previous updatedAt (not all reset to now)
5. Verify the created-at text is small, centered, and muted on each card
